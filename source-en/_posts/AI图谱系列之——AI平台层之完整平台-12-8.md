## 二 AI平台层

### 2.1 平台层理解

```
1.这一层可以理解为
“把模型、Agent、工具、推理能力全部打包成可用服务的平台”

2.本质上是：
AI能力 → 平台化 → 产品化 → 商业化
```

### 2.2 平台分类

```
- MCP平台
- Agent平台
- 应用平台
- 模型平台
- 推理平台
- Serverless AI平台
```

## 三 完整平台分层

```
应用层：
- AI产品（Notion / v0）

平台层：
- Dify / OpenClaw / LobeHub

能力层：
- LLM / RAG / Agent

推理层：
- Groq / vLLM

基础设施：
- GPU / Serverless（Modal）
```

## 四 核心总结

```
一句话：
AI平台 = 把“模型能力”变成“可用服务”
```

## 五 平台分工

|    层级    |   作用   |
| :--------: | :------: |
|  MCP平台   | 工具连接 |
| Agent平台  | 自动执行 |
|  应用平台  | 快速开发 |
|  模型平台  | 模型管理 |
|  推理平台  | 高速运行 |
| Serverless | 部署运行 |

## 六 2026趋势

```
平台化成为主流（不再手写AI系统）
Agent平台崛起（自动执行任务）
Serverless AI爆发（成本优化）
多模型 + 多平台协同
MCP成为事实标
```
