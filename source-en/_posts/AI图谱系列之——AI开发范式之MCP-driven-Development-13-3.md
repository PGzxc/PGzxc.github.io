## 二 AI开发范式

### 2.1 总体理解

```
传统开发：人写代码 → 调API → 完成功能
AI开发：人描述目标 → AI规划 → AI执行 → 人做决策
```

### 2.2  本质变化

```
“编写代码” → “编排智能”
```

## 三 MCP-driven Development(协议驱动开发)

### 3.1  核心思想

```
所有能力都通过 MCP 接入
```

### 3.2 架构

```
LLM
 ↓
MCP Client
 ↓
MCP Server（工具/API/系统）
```

### 3.3 优势

```
标准化工具接入
可扩展性强
解耦系统
```

### 3.4 趋势

```
MCP ≈ AI时代的 HTTP / USB
```
