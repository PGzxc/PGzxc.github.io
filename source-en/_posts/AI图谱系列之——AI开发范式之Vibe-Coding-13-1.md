## 二 AI开发范式

### 2.1 总体理解

```
传统开发：人写代码 → 调API → 完成功能
AI开发：人描述目标 → AI规划 → AI执行 → 人做决策
```

### 2.2  本质变化

```
“编写代码” → “编排智能”
```

## 三 Vibe Coding(感觉驱动开发)

### 3.1 理解

```
写代码 → 改成 → “描述感觉”
```

### 3.2 核心思想

```
不强调严谨设计
强调快速试错（Trial & Error）
人通过“感觉”不断调整
```

### 3.3  示例

```
“做一个像小红书一样的页面，
风格简洁一点，有卡片感”

AI直接生成 UI + 代码
```

### 3.4 适用场景

```
快速原型
创意产品
UI/前端开发
```

### 3.5 局限

```
可维护性差
架构容易混乱
```
