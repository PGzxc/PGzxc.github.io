## 二 完整闭环

```
输入 → 理解 → 检索 → 决策 → 执行 → 输出 → 记忆 → 再优化
```
## 三 AI总体架构(从请求到闭环)

### 3.1 总体架构

```
用户需求（Prompt）
↓
LLM（模型理解）
↓
RAG / Search（获取知识）
↓
Agent（决策与规划）
↓
Workflow（执行流程）
↓
Tool / Skill（调用能力）
↓
执行（API / UI / System）
↓
输出结果
↓
Memory（记忆更新）
```

### 3.2 架构分层图

```
用户层：
- Prompt / 多模态输入

模型层：
- LLM（推理核心）

数据层：
- RAG / Search

决策层：
- Agent

执行层：
- Workflow / Tool

动作层：
- API / Computer Use

输出层：
- Response

记忆层：
- Memory（闭环）
```

## 四 和传统架构对比

| 维度 | 传统系统 |    AI系统    |
| :--: | :------: | :----------: |
| 核心 | 业务逻辑 |     LLM      |
| 输入 |   参数   |    Prompt    |
| 数据 |    DB    |   RAG + DB   |
| 执行 |   代码   | Agent + Tool |
| 输出 |   固定   |   动态生成   |
| 进化 |  人维护  |    自学习    |

## 五 企业级扩展

```
1.API网关层（第10章）
-统一调用模型
-限流 / 计费

2.推理层（第6章）
-vLLM / TensorRT-LLM
-推理加速

3.平台层（第12章）
Dify / LobeHub / OpenClaw

4.安全层（重点）
- Prompt Injection防护
- 数据安全
- 权限控制
```

## 六 一句话

```
AI系统 = LLM + 数据 + Agent + 工具 + 执行 + 记忆
```

## 七 本质理解

```
AI系统不是“一个模型”，
而是一个“能思考、能调用工具、能执行任务、能持续优化”的智能系统。
```

## 八 面试终极回答

```
如果面试官让你讲 AI 系统架构，你可以这样说：

一个完整的AI系统，从用户输入开始，
通过LLM进行理解和推理，
结合RAG获取外部知识，
由Agent进行任务规划，
通过Workflow调用各种工具执行，
最终输出结果，并通过Memory形成闭环优化。
```
