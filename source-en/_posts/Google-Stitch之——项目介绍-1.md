## 二 项目介绍

### 2.1 官网地址

```
Google Stitch 官网：https://stitch.withgoogle.com/
```

### 2.2 介绍

```
Google-Stitch是 Google Labs 推出的 AI 原生 UI 设计工具，
核心口号是“从想法到 App”(From idea to app)。

它利用 Gemini 模型(支持 Gemini 2.5 Flash 快速模式和 Gemini 2.5 Pro 高保真实验模式)的多模态能力，
让任何人通过自然语言提示(text prompt)、图片、草图甚至 URL，就能快速生成高保真移动端和 Web 端 UI 设计，
并输出交互原型、前端代码和设计系统。
```

## 三 主要特点与价值

### 3.1 AI 驱动的“Vibe Design”

```
用描述性语言(如“现代简约的健身追踪 App 登录页，深色模式，带有渐变按钮”)
或上传参考图片，就能瞬间生成完整界面

支持无限画布(infinite canvas)、迭代修改和设计代理(design agent)。
```

### 3.2 支持输入方式

```
文本提示、图片上传(手绘 wireframe、截图、参考设计)、URL 提取设计系统、语音命令。
```

### 3.3 输出能力

```
高保真 UI 组件
交互原型
完整设计系统(DESIGN.md)
前端代码(通常为 HTML + Tailwind CSS，或导出到 Google AI Studio / Figma)。
```

### 3.4 协作与集成

```
无限画布支持添加图像、文本、代码；
可导出到 Figma、Google AI Studio；
支持与 AI 编码工具(如 Cursor、Claude)联动；设计代理能跟踪整个项目进度。
```

### 3.5 目标用户

```
设计师、开发者、产品经理、创业者、学生等。

特别适合快速 ideation、原型验证、从 0 到 1 搭建界面，无需专业设计技能也能高效产出。
```

### 3.6 免费使用

```
目前通过 Google 账号登录即可使用，无需等待列表或付费(属于 Google Labs 实验项目)
```

## 四 总结

```
Stitch 的核心优势在于打通设计与开发流程，
大大缩短从想法到可交互原型的时间，适合移动 App、Web 应用、内部工具、 landing page 等场景。

它不是替代 Figma，而是 AI 加速器，能快速生成起点，再精细打磨。
```
