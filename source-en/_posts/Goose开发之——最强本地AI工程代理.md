## 二 工具定位

```
1.定位
- 类型：AI 工程代理（Autonomous Agent）
- 开发方：Block / Linux Foundation
- 官网地址：https://github.com/block/goose

2.Goose 的核心理念：
-AI 自己写代码、运行、调试、修复
```

## 三 核心特点

### 3.1 完全本地运行(隐私拉满)

```
1.数据不出本机
2.支持：
  - Ollama
  - 本地 LLM
  - 云模型（可选）
```

### 3.2 自主执行能力

```
Goose 可以：
- 写代码
- 运行程序
- 执行 Shell
- 调用 Git
- 自动 debug

类似“AI 程序员”
```

### 3.3 MCP 工具生态

```
支持：
- 文件系统
- 数据库
- 浏览器（自动操作网页）
- API 调用

MCP = AI 工具插件系统
```

### 3.4 自动修复能力

```
流程：
写代码 → 运行 → 报错 → 读取日志 → 自动修复 → 再运行
```

### 3.5 完全免费开源

```
- Apache 2.0
- 无限制使用
```

## 四 安装与启动

### 4.1 安装

```
git clone https://github.com/block/goose
cd goose
npm install
```

### 4.2 启动

```
goose start
```

## 五 实战演示

### 5.1 场景：从 0 开发一个 Web 服务

```
1：输入需求:
创建一个 Node.js API，包含用户登录和 JWT 验证


2：Goose 自动执行

它会：
-创建项目
-写代码
-安装依赖
-启动服务

3：自动修复错误

如果报错：Error: module not found
Goose 会：
- 自动安装缺失包
- 修改代码
- 重启服务
```

### 5.2 最终结果

```
一个可运行的完整项目
```
