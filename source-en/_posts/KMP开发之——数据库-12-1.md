## 二 KMP 中数据库的核心挑战

### 2.1 平台差异

|  平台   |    原生数据库     |
| :-----: | :---------------: |
| Android |   Room / SQLite   |
|   iOS   | CoreData / SQLite |
| Desktop |      SQLite       |
|   Web   |     IndexedDB     |

### 2.2 设计目标

```
表结构统一
DAO 接口统一
多端使用方式一致
支持离线 / 缓存 / 同步
```

## 三 KMP 数据库方案选型

### 3.1 官方 & 主流推荐

```
SQLDelight
```

### 3.2 为什么选 SQLDelight？

```
Kotlin 官方维护
原生支持 Multiplatform
SQL 即 Schema
类型安全（编译期校验）
Android / iOS / Desktop / JS 全支持
```

### 3.3 SQLDelight vs Room

|   对比项   | SQLDelight | Room |
| :--------: | :--------: | :--: |
|   跨平台   |     可     | 不可 |
|  SQL 可控  |     可     | 不可 |
| 编译期校验 |     可     |  可  |
|  iOS 支持  |     可     | 不可 |

## 四 使用教程

### 4.1 项目依赖配置(shared 模块)

1-依赖

```
plugins {
    id("app.cash.sqldelight") version "2.0.1"
}

sqldelight {
    databases {
        create("AppDatabase") {
            packageName.set("com.example.db")
        }
    }
}
```

2-依赖声明

```
kotlin {
    sourceSets {
        val commonMain by getting {
            dependencies {
                implementation("app.cash.sqldelight:runtime:2.0.1")
            }
        }

        val androidMain by getting {
            dependencies {
                implementation("app.cash.sqldelight:android-driver:2.0.1")
            }
        }

        val iosMain by getting {
            dependencies {
                implementation("app.cash.sqldelight:native-driver:2.0.1")
            }
        }

        val desktopMain by getting {
            dependencies {
                implementation("app.cash.sqldelight:sqlite-driver:2.0.1")
            }
        }
    }
}
```

### 4.2 定义数据库 Schema(.sq 文件)

1-创建表结构(commonMain/sqldelight/com/example/db/article.sq)

```
1-创建过程
CREATE TABLE article (
    id INTEGER NOT NULL PRIMARY KEY,
    title TEXT NOT NULL,
    link TEXT NOT NULL
);

selectAll:
SELECT * FROM article;

insert:
INSERT OR REPLACE INTO article(id, title, link)
VALUES (?, ?, ?);

deleteAll:
DELETE FROM article;

2-SQLDelight 会自动生成：
-Article
-ArticleQueries
```

### 4.3 数据库 Driver 初始化(expect / actual)

1-commonMain

```
expect class DatabaseDriverFactory {
    fun createDriver(): SqlDriver
}
```

2-Android actual

```
actual class DatabaseDriverFactory(private val context: Context) {
    actual fun createDriver(): SqlDriver {
        return AndroidSqliteDriver(
            AppDatabase.Schema,
            context,
            "app.db"
        )
    }
}
```

3- iOS actual

```
actual class DatabaseDriverFactory {
    actual fun createDriver(): SqlDriver {
        return NativeSqliteDriver(
            AppDatabase.Schema,
            "app.db"
        )
    }
}
```

### 4.4 创建数据库实例

```
class Database(driverFactory: DatabaseDriverFactory) {
    val db = AppDatabase(driverFactory.createDriver())
}
```

### 4.5 DAO 层封装(Repository)

```
class ArticleRepository(database: Database) {
    private val queries = database.db.articleQueries

    fun getAll(): List<Article> =
        queries.selectAll().executeAsList()

    fun insert(article: Article) {
        queries.insert(
            article.id,
            article.title,
            article.link
        )
    }

    fun clear() {
        queries.deleteAll()
    }
}
```
