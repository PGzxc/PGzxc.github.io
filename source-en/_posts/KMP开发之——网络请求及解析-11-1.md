## 二 KMP 网络层的核心难点

### 2.1 平台差异

|  平台   |     原生网络      |
| :-----: | :---------------: |
| Android | OkHttp / Retrofit |
|   iOS   |    URLSession     |
| Desktop |   CIO / OkHttp    |
|   Web   |       Fetch       |

说明：KMP 不能直接复用 Retrofit

### 2.2 设计目标

```
API 调用方式统一
JSON 解析统一
错误模型统一
可测试、可扩展
```

## 三 KMP 网络库选型

### 3.1 官方 & 主流推荐组合

```
Ktor Client + kotlinx.serialization
```

### 3.2 为什么选 Ktor？

```
-Kotlin 官方团队维护
-原生支持 Multiplatform
-Engine 可替换（Android / iOS / JVM / JS）
-拦截器、超时、日志齐全
```

### 3.3  常见 Engine 对应关系

|  平台   | Engine |
| :-----: | :----: |
| Android | OkHttp |
|   iOS   | Darwin |
| Desktop |  CIO   |
|   Web   |   JS   |

### 4.2 创建 Ktor Client(核心封装)

1- 网络客户端统一入口

```
object HttpClientProvider {

    val client = HttpClient {
        install(ContentNegotiation) {
            json(
                Json {
                    ignoreUnknownKeys = true
                    isLenient = true
                }
            )
        }

        install(HttpTimeout) {
            requestTimeoutMillis = 15_000
        }
    }
}
```

2-说明

```
这一层只关心“网络能力”，不关心业务
```

### 4.3  发起网络请求(GET / POST)

1-GET 请求

```
suspend inline fun <reified T> get(url: String): T {
    return HttpClientProvider.client.get(url).body()
}
```

2- POST 请求

```
suspend inline fun <reified T> post(
    url: String,
    params: Any
): T {
    return HttpClientProvider.client.post(url) {
        setBody(params)
    }.body()
}

说明：使用 reified 实现 类型安全解析
```

### 4.5 错误处理统一模型

1-定义 Result 封装

```
sealed class NetworkResult<out T> {
    data class Success<T>(val data: T) : NetworkResult<T>()
    data class Error(val message: String) : NetworkResult<Nothing>()
}
```

2-包装请求

```
suspend inline fun <reified T> safeGet(
    url: String
): NetworkResult<T> {
    return try {
        NetworkResult.Success(get(url))
    } catch (e: Exception) {
        NetworkResult.Error(e.message ?: "Network error")
    }
}
```

### 4.6 ViewModel 中的使用方式(KMP)

1-代码

```
class ArticleViewModel {

    private val _state =
        MutableStateFlow<UiState<List<Article>>>(UiState.Loading)

    val state = _state.asStateFlow()

    fun load() {
        viewModelScope.launch {
            when (val result = safeGet<List<Article>>(API)) {
                is NetworkResult.Success ->
                    _state.value = UiState.Success(result.data)
                is NetworkResult.Error ->
                    _state.value = UiState.Error(result.message)
            }
        }
    }
}
```

2-说明

```
网络 → 状态 → UI
```

## 五 小结

```
KMP 网络层首选 Ktor Client
Engine 自动适配平台
kotlinx.serialization 负责解析
网络层只做“数据获取”
UI 层完全无感知平台差异
```
