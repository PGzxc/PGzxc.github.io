## 二 实战案例

### 2.1 实战1：用 Manus 做一个完整网站

```
1-目标
从 0 生成一个个人网站(可部署)

2-Prompt（可直接复制）
帮我生成一个个人网站：
要求：
- 包含首页、项目展示、关于我
- 响应式设计（适配手机）
- 风格简约现代
- 使用 HTML + CSS + JS
输出：完整网站项目文件

3-Manus 执行流程:
-设计页面结构
-生成 UI 布局
-编写前端代码
-组织项目结构
-输出网站文件

4-输出结果
/site
 ├── index.html
 ├── projects.html
 ├── about.html
 ├── styles.css
 └── script.js
 
5-如何部署
可直接部署到：

- GitHub Pages
- Vercel
- Netlify

完全不需要手写代码

6-进阶玩法
在网站中加入项目筛选功能 + 动画效果
```

### 2.2 实战2：数据分析项目

```
1-目标:
上传 CSV → 自动生成分析报告

2- Prompt（标准模板）
分析这个CSV文件：
要求：
-数据清洗
-核心指标统计
-生成可视化图表
-输出Excel报告 + 分析结论

3-执行流程
Manus 自动完成：

- 数据读取
- 清洗处理
- 分析计算
- 图表生成
- 报告输出

4-输出内容:
- 图表（趋势/柱状图）
- 分析报告
- Excel 文件

5-进阶玩法
按地区分析销售趋势，并预测未来走势

6-适合场景
- 电商数据分析
- 用户行为分析
- BI 报告生成
```

### 2.3 实战3：AI工具调研自动化

```
1-目标:
自动调研多个 AI 工具并生成对比报告

2-Prompt:
调研以下AI编程工具：
Cursor、Copilot、Codeium、Continue

要求：
-功能对比
-优缺点
-使用场景
-是否支持本地模型

输出：表格 + 总结报告

3-执行逻辑（官方 Wide Research）
- 多 Agent 并行调研
- 分别分析
- 汇总结果

4-输出结果
表格：工具、优点、缺点、场景

5-进阶玩法
扩展到20个AI工具，并按开发效率排序
```

## 三 组合案例(最强用法)

```

```

```

## 四 项目级最佳实践

### 4.1 模块化任务

```
不要一口吃：
数据分析 → 网站生成 → 自动化

分步骤更稳定
```

### 4.2 每一步都可复用

```
Manus 生成的：
- 代码 
- 模板 
- 报告 

都可以重复利用
```

```
