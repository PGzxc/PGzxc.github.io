## 二 文件处理(最常用能力)

### 2.1 支持文件类型(官方能力)

```
1.Manus 支持：
- CSV / Excel
- PDF
- TXT / Markdown
- 图片（部分解析）

2.官方说明：
Manus 可在 sandbox 环境中读取和处理文件
```

### 2.2 基础用法(上传 +1句话)

```
1.示例任务：
帮我分析这个CSV文件，找出销售最高的产品，并生成图表

2. Manus 会自动：

-读取文件
-清洗数据
-分析指标
-生成图表
-输出结果

无需写代码
```

### 2.3 高级用法(结构化输出)

```
1.分析该Excel数据：
-按地区统计销售额
-找出Top10产品
-输出为Excel报告

2.输出：
- 新 Excel 文件
- 含统计结果 + 图表
```

### 2.4 PDF 处理(非常强)

```
1.PDF处理
总结这个PDF的核心内容，并生成结构化摘要

2.Manus 会：
- 提取文本
- 分析结构
- 输出摘要 / 报告

3.适合：
- 论文
- 合同
- 技术文档
```

## 三 网站生成(开发者必用)

### 3.1 一句话生成网站

```
示例：帮我做一个个人博客网站，包含首页、文章列表、关于页面
```

### 3.2 Manus 实际会做

```
1.Manus会做：
- 设计页面结构
- 编写 HTML / CSS / JS
- 组织目录
- 输出完整项目

2.输出内容类似：
/website
  ├── index.html
  ├── about.html
  ├── styles.css
  └── script.js
```

### 3.3 进阶：指定技术栈

```
1.指定技术栈：
使用React帮我生成一个个人网站，包含项目展示

2.Manus 会：
- 生成 React 项目结构
- 写组件代码
- 输出完整工程
```

### 3.4 进阶：UI + 功能

```
1.要求：
做一个现代风格的产品展示网站，带响应式布局

2. 官方能力支持：
- UI设计
- 基础交互
- 响应式页面
```

## 四 自动化任务(核心价值)—这是 Manus 最强的地方

### 4.1 批量任务(官方 Wide Research)

```
1.示例：
调研50个AI产品，并整理成对比表

2.Manus 会：
- 并行处理多个任务
- 分别分析
- 汇总结果

3.传统 AI 做不到这一点
```

### 4.2 多步骤自动化

```
1.示例：
每天收集AI新闻，并生成摘要报告

2.执行流程：
-搜索数据
-筛选内容
-总结
-输出报告
```

### 4.3 数据 → 报告 自动化

```
1.示例：
分析这些销售数据，并生成周报

2.自动完成：

- 数据处理
- 可视化
- 报告生成
```

## 五 组合能力(真正进阶)—Manus 的强大在于“组合任务”

### 5.1 示例(完整工作流)

```
-读取这个CSV文件，
-分析销售数据，
-生成图表，
-最后做成一个网页展示结果
```

### 5.2 Manus 实际执行

```
1. 读取数据
2. 数据分析
3. 生成图表
4. 写网页代码
5. 输出网站

一次完成完整项目
```

## 六 常见问题(官方实践总结)

### 6.1 问题1：任务失败

```
1.原因：
- 描述不清
- 任务过大

2.解决：
拆成多个任务
```

### 6.2 问题2：结果不符合预期

```
优化方式：明确输出格式 + 细节要求
```

### 6.3 问题3：执行时间长

```
1.原因：
- 任务复杂
- 多步骤执行

2.Manus 是“执行型 AI”，不是即时回复
```
