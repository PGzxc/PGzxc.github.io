## 一 概述

```
本文介绍：
1.用户、群组和共享权限系统
2.通过它们实现：多人不同权限、安全共享
```

<!--more-->

## 二 核心概念

### 2.1 用户(Users)

```
实际登录账号(如家人、朋友)。
每个用户可独立设置密码和邮箱（邮箱用于云访问）
```

### 2.2 用户组(Groups)

```
将多个用户打包管理，权限统一赋给组更高效。
OS5 下组功能有所弱化。
```

### 2.3 共享(Shares)：NAS上的文件夹(如: 照片/电影/工作)

```
1.公共共享（Public）：默认所有人可见，通常设为只读。
2.私有共享（Private）：默认隐藏，只有明确授权的用户/组才能看到和访问。
```

### 2.4 权限级别(三种主流)

```
1.读/写（Read/Write）：查看 + 修改 + 删除 + 上传。
2.只读（Read Only）：仅查看和下载，不能改动。
3.无访问（No Access）：完全不可见（最安全）。
```

### 2.5 权限生效规则

```
用户最终权限 = 自身权限 与 所属所有组权限 的最严格交集。
示例：用户自己对某共享是“读/写”，但所属组是“无访问” → 最终无访问。
```

## 三 操作入口

```
用浏览器访问NAS仪表板（Dashboard）：

1.输入 http://mycloudex2ultra.local/ 或设备IP地址。
2.用 admin 账号登录（建议立即设置强密码）。
```

## 四 完整设置流程

### 4.1 创建用户

```
1.左侧导航 → 用户（Users）。
2.点击右下角 +（添加用户）。
3.输入：用户名、姓名、邮箱（邮箱必填且唯一，用于云访问）。
4.设置密码（开启密码开关）。
5.下一页可选：加入已有用户组（可跳过）。
6.最后一步：在共享访问区域，为该用户直接设置对各个私有共享的权限（读/写 / 只读 / 无）。
7.点击应用或保存。
```

### 4.2 创建用户组(强烈推荐，多人场景省事)

```
1.在“用户”页面切换到用户组（Groups）标签（或直接点击Groups）。
2.点击 + Add Group。
3.输入组名（如“家庭组”“朋友组”“只读组”）。
4.创建后选中该组。
5.在下方共享访问区域，统一设置该组对所有私有共享的权限。
以后新增用户只需加入对应组，即自动继承权限。
```

### 4.3 创建私有共享并授权

```
1.左侧导航 → 共享（Shares）。
2.点击 + Add Share 创建新文件夹。
3.重要：取消“Public（公共）”勾选，改为私有。
4.创建完成后，选中该共享。
5.向下滚动到User Access 或 权限设置：
 -可直接给单个用户授权。
 -或给用户组授权（推荐）。
```

## 五 实用场景示例

### 5.1 全家照片共享

```
创建私有共享“照片” → 给“家庭组”设读/写 → 给“朋友组”设只读 → 其他人无访问。
```

### 5.2 小孩专用账号

```
新建用户“小明” → 只加入“只读组” → 只给他“动画片”“儿歌”共享读/写权限，其他共享全部无访问。
```

### 5.3 公司/工作资料保护

```
创建私有共享“工作” → 只给“工作组”读/写 → 其他组/用户全部无访问。
```

## 六 常见问题快速排查

### 6.1 设置权限后仍“拒绝访问”或看不到共享？

```
检查用户是否属于多个组，其中某个组设为“无访问”。
私有共享必须至少有一个“读”以上权限给用户或其组。
Windows映射驱动器时，用NAS用户名/密码登录（非Windows账号）。
```

### 6.2 公共共享想让admin可写、别人只读？

```
公共共享无法针对admin单独设写权限（系统限制）。
解决：把原Public改为私有 → admin设读/写 → 其他人/组设只读。
```

### 6.3 手机APP云访问看不到文件？(OS 5常见)

```
云用户必须至少有一个共享有读权限。
邮箱必须唯一且验证通过。
OS 5下，云用户不支持加入组（需直接给用户设权限）。
共享需开启“Mobile & Web App Access”。
```
