## 二 IP地址

### 2.1 什么是 IP 地址？

```
一、概念：
-每个设备都有一个“地址”：
-类似：身份证 / 门牌号


二、两种 IP（必须记住）

2.1、内网 IP（私有 IP）

示例：
192.168.x.x
10.x.x.x
172.16.x.x

特点：
-只能在局域网使用
-不能被外网访问

2.2、公网 IP

示例：
1.2.3.4
8.8.8.8

特点：
-全球唯一
-可以被外网访问
```

### 2.2 为什么你没有公网 IP？

```
1. 因为：
IPv4 地址不够用

2.解决方案：
多个设备共享一个公网 IP

3.这就是：
NAT（网络地址转换）
```

## 三 NAT 是什么？

### 3.1 定义

```
把内网 IP 转换成公网 IP
```

### 3.2 工作过程

```
1.工作过程

你电脑（192.168.1.10）
       ↓
路由器（公网IP：1.2.3.4）
       ↓
    互联网
    
2.实际访问：
服务器看到的是：1.2.3.4（不是你电脑）
```

### 3.3 NAT 做了什么？

```
1.维护一张表：
内网IP:端口 → 公网IP:端口

2.举例：
访问时：192.168.1.10:1234 → 1.2.3.4:5678
返回时：1.2.3.4:5678 → 192.168.1.10:1234

3.这样就实现了：
-内网访问外网
```

## 四 遇到问题

### 4.1 为什么外网访问不到你？

```
1.因为：
NAT 只允许“已建立连接”的返回流量

2.外网主动访问：
1.2.3.4 → ???  不知道去哪

3.路由器：
没有记录 → 丢弃

4.所以：
外网无法主动访问内网
```

### 4.2 现实问题：很多人没有公网 IP

```
1. 特别是：
- 家庭宽带
- 移动网络

2.你其实在：
内网中的内网（CGNAT）

3. CGNAT 结构：
你 → 运营商内网 → 公网

4.结果：
你连端口映射都做不了
```

## 五 解决方案

### 5.1 解决方案1：端口映射(Port Forwarding)

```
一、原理：
手动告诉路由器：某个端口 → 转发到某台设备

二、示例：

2.1 访问图示
公网：1.2.3.4:8080
   ↓
内网：192.168.1.100:80

2.2 效果：
访问 1.2.3.4:8080 → 访问 NAS

三、优点：

-简单
-不需要额外服务器

四、 缺点：

-需要公网 IP
-有安全风险
-被运营商限制
```

### 5.2 解决方案二：内网穿透(核心)

```
1.思路：
内网设备 → 主动连接公网服务器

2.架构：

内网设备（你家）
        ↓
公网服务器（VPS）
        ↓
    外网用户

3. 这样：
-绕过 NAT
-实现访问
```

## 六 知识补充

### 6.1 端口映射 vs 内网穿透

|      对比      | 端口映射 | 内网穿透 |
| :------------: | :------: | :------: |
| 是否需要公网IP |   需要   |  不需要  |
|      难度      |    低    |    中    |
|     安全性     |   较低   |   较高   |
|      推荐      |  不推荐  |   推荐   |

### 6.2 结合 VPN

```
1.VPN（如 WireGuard）可以：
-打通网络
-建立“虚拟局域网”

2.结构：
手机 → VPN → 家里 NAS

3.本质：
-不需要穿透
-直接变“同一网络”
```

## 七 总结图

```
1.没公网IP：外网 → 内网(无法实现)
2.端口映射：外网 → 路由器 → 内网
3.内网穿透：内网 → VPS → 外网
4.VPN：设备 ↔ 虚拟内网
```
