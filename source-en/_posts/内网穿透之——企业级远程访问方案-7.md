## 二 相关概念

### 2.1 解决什么问题？

```
1.当系统从“家庭 NAS”升级到“企业/团队/多服务系统”时，会遇到新问题：

-VPN 一开就是“全网可见”（不安全）
-端口映射太危险（暴露攻击面）
-frp 虽然好用，但缺少“访问控制体系”
-多人访问权限混乱
-无法审计谁访问了什么

2.这时候就需要：
零信任（Zero Trust）+ 内网穿透 + 访问控制体系
```

### 2.2 什么是零信任(Zero Trust)

```
1. 核心思想：
 -不再默认“内网是安全的”
 -每一次访问都必须验证

2.传统网络
 -进入内网 = 全部信任
 -连上 VPN = 直接访问所有服务
 -一旦账号泄露 = 全盘沦陷

3.零信任网络
 -每个请求 = 单独验证
 -访问 NAS → 先认证身份
 -访问 Docker → 再次鉴权
 -每个服务独立权限控制
```

## 三 企业级远程访问架构-标准架构图

```
        🌍 用户（手机 / PC）
                 ↓
        🔐 Zero Trust Gateway
      （身份认证 + 访问控制）
                 ↓
        🔁 反向代理层（Nginx）
                 ↓
   ┌──────────────┬──────────────┐
   ↓              ↓              ↓
 NAS服务       API服务        Docker服务
（文件）     （后端接口）   （应用集群）
```

## 四 核心组件拆解

### 4.1  Zero Trust Gateway(核心入口)

```
1.推荐方案：
-Cloudflare Zero Trust
-Cloudflare Tunnel
-或自建 OAuth2 Proxy

2.功能：
-身份认证（Google / GitHub / OTP）
-设备信任管理
-IP / 地区限制
-MFA（多因素认证）
```

### 4.2 内网穿透层(替代传统 frp)

1-推荐

```
Cloudflare Tunnel（最简单）
frp（可控性更高）
```

2-对比

|       方案        | 是否需要公网IP | 安全性 | 运维难度 |
| :---------------: | :------------: | :----: | :------: |
|        frp        |     不需要     | 4颗星  |  3颗星   |
| Cloudflare Tunnel |     不需要     | 5颗星  |  1颗星   |

### 4.3 反向代理层(Nginx)

```
1.作用：

-统一入口
-路由分发
-HTTPS终止

2.示例：

nas.example.com → NAS
api.example.com → 后端
media.example.com → 影音系统
```

### 4.4 内部服务层(NAS / Docker)

```
典型服务：

-Synology DiskStation
-Jellyfin
-Home Assistant
-Git / CI / API服务
```

## 五 两种企业级架构方案

### 5.1  方案 A：Cloudflare 零信任(推荐)

```
1.架构：
用户 → Cloudflare Access → Tunnel → 内网服务

2.优点：
-不需要 VPS
-不需要公网IP
-自动HTTPS
-登录认证体系完整

3.适合：
-个人开发者
-小团队
-NAS远程访问
-内部工具系统
```

### 5.2 方案 B：VPS + frp + Zero Trust(进阶)

```
1.架构：
用户
 ↓
Zero Trust Gateway
 ↓
VPS（frps）
 ↓
frpc（内网）
 ↓
NAS / Docker

2.优点：
-完全可控
-可扩展
-支持复杂企业网络

3.缺点：
-配置复杂
-需要维护服务器
```

## 六 进阶

### 6.1 权限控制模型

```
1.RBAC（角色权限控制）
 -Admin → 全部权限
 -User → 文件访问
 -Guest → 只读访问

2.ABAC（属性权限控制）

设备类型 + 地理位置 + 时间

例：
-只能公司 WiFi 访问
-晚上禁止访问 NAS
```

### 6.2 企业级安全增强

```
1.多因素认证（MFA）
-Google Authenticator
-邮箱 OTP
-硬件密钥（Yubikey）

2.TLS 全链路加密
用户 → HTTPS → Gateway → HTTPS → NAS

3.审计日志

记录：
-谁访问了 NAS
-访问了什么文件
-下载了什么内容

4.最小权限原则
-NAS 管理端禁止公网访问
-Docker API 只允许内部调用
```

## 六 常见误区

### 6.1 误区1：VPN就是安全的

```
错误:
VPN = “进入内网钥匙”
但没有权限控制
```

### 6.2 误区2：端口映射简单又好用

```
错误:
等于：“把家门直接开到马路上”
```

### 6.3  误区3：frp已经足够

```
半对:
frp ≠ 安全系统
它只是“通道”
```

## 七 推荐最终架构(个人/企业通用)-最佳组合

```
Cloudflare Zero Trust（入口）
        ↓
Cloudflare Tunnel / frp（穿透）
        ↓
Nginx（统一入口）
        ↓
NAS / Docker / API服务
```

## 八 总结

### 8.1 核心一句话

```
内网穿透解决“能访问”，零信任解决“能不能访问”
```

| :----: | :----------: |
| 连接层 | frp / Tunnel |
| 网关层 |    Nginx     |
