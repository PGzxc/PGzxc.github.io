## 二 什么是内网穿透

### 2.1 什么是“内网穿透”？

```
让“外网”访问“内网设备”
```

### 2.2 为什么访问不到？

```
1.你的设备其实在“内网”

   你家电脑/NAS
       ↓
     路由器
       ↓
   公网（互联网）
   
2.关键问题：
外网无法主动访问你家设备

3.真实例子：
你家 NAS：IP：192.168.1.100
外网访问：http://192.168.1.100 (无效)
为什么？因为：192.168.x.x = 内网地址（私有IP）
```

## 三 内网穿透理解

### 3.1 什么是 NAT？(核心理解)

```
1.NAT = 地址转换
路由器在做这件事：内网设备 → 共享一个公网IP上网

2.实际过程：
你电脑（192.168.1.10）
       ↓
路由器（公网IP：1.2.3.4）
       ↓
    互联网

3.结果：
外网只能看到：1.2.3.4（你的路由器）
看不到：192.168.1.10
```

### 3.2 内网穿透的本质(重点)

```
1.解决方案：
让内网设备“主动连接”外网服务器

2.架构：

内网设备
   ↓（主动连接）
公网服务器（中转）
   ↓
外网用户

3.核心思想：
反向连接（Reverse Connection）
```

### 3.3 类比理解

1-类比打电话：

|     场景     |      类比      |
| :----------: | :------------: |
| 外网访问内网 | 陌生人打你电话 |
|   内网穿透   | 你主动打给对方 |

2-所以

```
必须由内网“主动发起连接”
```

### 3.4 内网穿透 vs VPN

1-对比

|   对比   |   内网穿透   |     VPN      |
| :------: | :----------: | :----------: |
|   目的   | 外网访问内网 | 建立专用网络 |
| 连接方式 |     单向     |     双向     |
| 使用场景 | 远程访问 NAS |   全局网络   |

2-实际组合：

```
1.推荐使用：
WireGuard（VPN） + 内网穿透

2.为什么？
- VPN：打通网络
- 穿透：解决 NAT
```

## 四 解决方案

```

2.远程桌面

3.自建服务

- API 服务

本地服务给外网访问：localhost:3000 → 公网URL
```

## 五 总结

### 5.1 核心技术—内网穿透(3个核心)

```
-NAT（问题）
-反向连接（核心）
-中转服务器（关键）
```

### 5.2 本篇总结

```
-内网设备默认无法被外网访问
-NAT 是根本原因
-内网穿透通过“反向连接”解决
```
