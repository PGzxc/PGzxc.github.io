## 二 内网穿透工具分类

### 2.1 三大类方案

```
1.自建型（frp）
2.SaaS型（ngrok）
3.虚拟网络型（Zerotier / VPN）
```

### 2.2 本质区别

|   类型   |     核心     |
| :------: | :----------: |
|   frp    | 自己搭服务器 |
|  ngrok   | 用别人服务器 |
| Zerotier | 建虚拟局域网 |

## 三 三类工具介绍

### 3.1 frp(最常用)

```
1.是什么？
一个开源内网穿透工具

2.工作原理：

内网设备（frpc）
      ↓
公网服务器（frps）
      ↓
  外网访问

3.优点：
-完全自控
-免费
-性能好
-支持多协议（HTTP / TCP / UDP）

4.缺点：
-需要 VPS
-需要自己配置

5.适合人群：
- 会用 Linux
- 有 VPS（比如之前的 DMIT）
- 想长期稳定使用
```

### 3.2 ngrok(最简单)

```
1.是什么？
一键穿透工具（无需服务器）

2.原理：
你电脑 → ngrok服务器 → 公网URL


3.优点：

-极其简单
-适合新手
-立即可用

4.缺点：
-免费版限速
-URL 会变
-不稳定

5.适合：
临时使用：
- Demo
- Webhook
- 调试
```

### 3.3 ZeroTier(高级)

```
1.是什么？
虚拟局域网工具（类似 VPN）

2.原理：
设备A ↔ Zerotier网络 ↔ 设备B
直接变：像在同一个局域网

3.优点：
-不需要公网 IP
-自动组网
-安全性高

4. 缺点：
-需要安装客户端
-不适合开放公网服务

5.适合：
- 远程访问 NAS
- 远程办公
- 内网设备互通
```

## 四 工具对比及选择

### 4.1 三者核心对比

|     对比     |  frp  | ngrok  | Zerotier |
| :----------: | :---: | :----: | :------: |
|   是否自建   |  是   |  不是  |   不是   |
|     难度     |  中   |   低   |    中    |
| 是否需要 VPS | 需要  | 不需要 |  不需要  |
|    稳定性    | 5颗星 | 2颗星  |  4颗星   |
| 是否开放公网 |  是   |   是   |   不是   |
|   推荐指数   | 5颗星 | 2颗星  |  4颗星   |

### 4.2 如何选择？

```
场景一：条件：有 VPS/会 Linux/想长期用
选择：frp（最佳方案）

场景二：临时调试
选择：ngrok

场景三：远程访问 NAS
选择：Zerotier 或 WireGuard

场景四：企业 / 稳定
组合：WireGuard + frp + Xray
```

## 五 进阶

### 5.1 推荐架构

```
1.已有技术：
- VPS
- Xray
- Clash

2.最优方案：

NAS（内网）
   ↓
frp client
   ↓
VPS（frp server）
   ↓
公网访问

3. 同时：
Clash → Xray → 外网

4. 形成完整系统：
代理 + 穿透 一体化
```

### 5.2 重要认知

1-说明

```
内网穿透 ≠ 代理
```

2-对比

| 功能 |     说明     |
| :--: | :----------: |
| 代理 |   访问外网   |
| 穿透 | 让外网访问你 |

### 5.3 选哪个

```
1.推荐：
-frp（主）
-WireGuard（辅助）

2. 不推荐：
ngrok（不适合长期）
```

## 六 总结

```
1.frp = 最强自建方案
2.ngrok = 临时工具
3.Zerotier = 虚拟内网
```
