## 一 概述

```
本文介绍：
 1.Basic 模式不能直接“原地转换”升级到 SHR 模式
 2.想用 SHR，只能重建存储池 + 迁移数据
```

<!--more-->

## 二 Basic升级到SHR方案

### 2.1 最安全方案：新建 SHR 池 + 迁移数据(推荐，零风险)

适合你有空盘位或额外硬盘的情况(比如1821+有8位，现在只用了2位)。

1-过程

```
1.备份数据(必须先做)
-用 Hyper Backup 把 Basic 卷的所有共享文件夹备份到移动硬盘、另一个卷、或云(Google Drive)。
-或者用 File Station 内部复制到另一个 Basic 卷（如果你有空盘）。
-重要数据双备份，别偷懒。

2.删除旧 Basic 存储池
-Storage Manager → 存储 → 选你的Basic存储池 → 动作 → 删除存储池(会删卷和数据，但你已备份)。
-确认删完，硬盘变“未初始化”。

3.创建新 SHR 存储池
-Storage Manager → 创建 → 创建存储池 → 手动创建。
-RAID 类型：选 SHR（或 SHR-1）。
-选硬盘：先选你原来的那块硬盘+至少1块新/空硬盘(SHR至少2盘起步才有保护；单盘 SHR跟 Basic 差不多，没保护)。
-硬盘大小混用也没问题。
-应用 → 系统会初始化和检查（后台跑，几小时到1天）。

4.创建新卷
-存储池建好后 → 创建卷 → 选新 SHR 池 → 文件系统 Btrfs(推荐)→ Max 容量 → 应用。

5.迁移数据回来
-用 File Station 内部复制(或 Shared Folder Sync / Hyper Backup 恢复)。
-拷完检查文件完整性(大小、MD5 可选)。
-删旧备份(如果空间紧)。

6.可选：加更多盘扩容
-以后加盘：存储池 → 动作 → 添加硬盘 → 选新盘 → 系统重建(后台，边用边修)。
-换大盘：替换硬盘 → 插新盘 → 自动重建。
```

2-优缺点

```
1.耗时：备份+迁移是主力(电影29TB，内部复制快；网络备份慢点)，总1-3天。
2.优点：零数据丢失风险，以后 SHR 超灵活（混盘、易扩容、1盘保护）。
3.缺点：需要至少1块额外硬盘起步 SHR。
```

### 2.2 备选方案：如果没空盘，先转 RAID 1(有限制)

```
1.如果你正好有2块大小一样的硬盘(比如两个8T)，可以把 Basic 转 RAID 1(镜像)。
2.操作：Storage Manager → 存储池 → 动作 → 更改 RAID 类型 → 选 RAID 1 → 加第二块盘 → 系统重建。
3.但转 SHR 还是不行（RAID 1 也不能直接转 SHR，除非重建）。
4.社区说：RAID 1 和 SHR-1 在2盘时功能一样，但 SHR 以后扩容更灵活
```
