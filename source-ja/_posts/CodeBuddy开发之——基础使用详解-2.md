## 二 AI 对话模式

### 2.1 说明

```
CodeBuddy 的核心交互方式是 AI 对话驱动开发
```

### 2.2 打开方式

```
- Windows：Ctrl + I
- Mac：Cmd + I
```

### 2.3 基础用法

```
1.基础用法
解释这段代码的作用
帮我优化这段代码性能
帮我设计一个用户登录系统

2.支持：
- 代码解释
- Bug 修复
- 架构设计
- 文档生成
```

### 2.4 官方建议

```
Prompt 要明确 + 有上下文
避免模糊描述
```

## 三 上下文引用机制(核心能力)

CodeBuddy 最大优势之一：上下文感知能力

### 3.1 @ 引用文件

```
1.说明：
@UserService 帮我优化这个类

2.AI 会：
- 自动读取文件
- 理解上下文
- 给出针对性修改
```

### 3.2 引用多个文件

```
1.示例
@UserController @UserService 帮我分析调用链

2.适用于：

- 复杂业务分析
- Bug 定位
```

### 3.3 引用文件夹

```
@/auth-module 解释这个模块架构
```

### 3.4 当前代码上下文

```
在编辑器中选中代码：优化这段代码

自动带上下文
```

### 3.5 官方设计理念

```
让 AI 理解你的整个工程，而不是单文件
```

## 四 Prompt 编写技巧(高效使用关键)

### 4.1 错误写法

```
帮我写代码

太模糊 → 结果不稳定
```

### 4.2 推荐写法(结构化 Prompt)

```
用 Spring Boot 实现一个登录接口：
要求：
1. 使用 JWT
2. 包含参数校验
3. 返回统一 JSON 格式
```

### 4.3 模板(强烈推荐)

```
任务：
实现一个XXX功能

技术栈：
Spring Boot + MySQL

要求：
1. xxx
2. xxx
3. xxx
```

### 4.4 进阶 Prompt(结合上下文)

```
@UserService
帮我：
1. 优化代码结构
2. 提升可读性
3. 增加异常处理
```

### 4.5 官方建议总结

```
- 明确任务
- 给约束条件
- 指定技术栈
- 指定输出格式
```

## 五 进阶

### 5.1 代码编辑模式(Edit)—不只是“问”，还能“改”

```
一、用法
1.在代码中写：将同步代码改为异步实现

2.CodeBuddy 会：
 - 直接修改代码
 - 给出 diff
 
二、常见场景

- 重构代码
- 增加日志
- 添加异常处理
- 改写为另一种写法
```


### 5.2 Agent 模式(自动执行任务)

```
1.CodeBuddy 支持 Agent 工作模式：
AI 自动完成多步骤任务

2.示例

帮我实现一个用户系统（包含登录注册）

自动执行：
-创建目录
-生成代码
-补全逻辑
-修复错误

3.注意
- Agent 会修改多个文件
- 建议在 Git 下使用
```

### 5.3 对话上下文管理

```
1.会话隔离

每个对话：
- 独立上下文
- 不互相污染

2.多会话场景
- 一个做后端
- 一个做前端
- 一个调试 Bug

3.历史记录
- 可回溯
- 可复用 Prompt
```

### 5.4 常用高效指令(官方推荐)

```
1.代码生成：生成一个 RESTful API
2.Bug 修复：@OrderService 找出潜在问题
3.性能优化：优化这段代码的性能
4.架构分析：@/project 解释项目结构
5.单元测试：为这个类生成单元测试
```

## 六 新手避坑指南

### 6.1 问题1：AI答非所问

```
1.原因：
- Prompt 太模糊

2.解决：
- 加限制条件
```

### 6.2 问题2：代码不符合项目结构

```
解决：@项目目录 按当前结构生成代码
```

### 6.3 问题3：生成代码不可用

```
解决：修复并确保代码可运行
```

## 七 本篇总结

```
1.CodeBuddy 的高效使用关键：
上下文 + Prompt = 生产力

2.核心三点：
-会用 `@`（上下文）
-会写 Prompt（结构化）
-善用 Agent（自动化）
```
