## 二 ComfyUI 是什么

### 2.1 ComfyUI 的官方定位

```
ComfyUI 是一个基于 节点(Node)与工作流(Workflow)的 AI 内容生成系统，
最初用于 Stable Diffusion 图像生成，
目前已扩展为一个通用多模态 AI 工作流引擎。

它支持的方向包括但不限于：
-图像生成（Image）
-视频生成（Video）
-音频生成（Audio）
-3D 模型（3D Model）
-大语言模型（LLM）
-工具型流程（Utility）
-云端 AI 服务（Partner Nodes）
```

### 2.2 正确理解ComfyUI

```
很多人第一次打开 ComfyUI 会懵，因为它不是传统软件界面。

一个正确的理解方式是：ComfyUI ≈ AI 领域的“可视化数据流编程工具”
```

### 2.3 ComfyUI 与传统 AI 工具的本质区别

```
1-A1111（WebUI）
-面板式
-参数堆叠
-流程隐藏在代码中

2-Midjourney
-指令式
-黑盒
-不可控

3-ComfyUI
-流程显式
-数据可追踪
-每一步都可修改

4-一句话总结：A1111 是“用”，ComfyUI 是“搭”
```

### 2.4 什么是 Node(节点)

```
1、在 ComfyUI 中：
一个 Node = 一个明确的功能
比如：加载模型、输入提示词、采样、解码、保存图片

2、节点只有三件事：输入、处理、输出
```

### 2.5 什么是 Workflow(工作流)

```
Workflow = 多个节点按顺序连接形成的完整流程
例如一个最简单的文生图流程：提示词 → 模型 → 采样 → 解码 → 输出图片
ComfyUI 把这个流程完全可视化了。
```

## 三 ComfyUI 适合哪些人

### 3.1 非常适合的人群

```
1.AI 深度使用者
-不满足于“出图”
-想控制画面结构、风格、稳定性

2.设计师 / 插画师
-需要可复现的工作流
-需要统一风格批量生成

3.开发者 / 工程师
-理解数据流
-想把 AI 接入系统
-自动化生成内容

4.内容生产者
-批量生成封面、配图、视频
-希望降低重复劳动
```

### 3.2 不太适合的人群

```
-只想一句话出图
-不愿意理解流程
-对参数和逻辑完全没兴趣
-对这类用户，Midjourney 更合适。
```

## 四 设备要求与性能认知

### 4.1 显卡是核心(推荐配置)

| 项目 |    建议     |
| :--: | :---------: |
| 显卡 | NVIDIA RTX  |
| 显存 |  8GB 起步   |
| 推荐 | 12GB / 16GB |

说明：ComfyUI ≠ 魔法  显存不够，流程再好也跑不动

### 4.2 不同用途的显存参考

|  使用场景  |  显存   |
| :--------: | :-----: |
|   文生图   |  6–8GB  |
|    SDXL    | 8–12GB  |
| ControlNet | 10–12GB |
|    视频    |  12GB+  |

### 4.3 CPU / 内存 / 磁盘

```
CPU：i5 / Ryzen 5 以上
内存：16GB（推荐 32GB）
磁盘：SSD（模型多，占用大）
```

### 4.4 操作系统建议

|  系统   | 评价 |
| :-----: | :--: |
| Windows | 5星  |
|  Linux  | 4星  |
|  macOS  | 2星  |

## 五 ComfyUI 安装方式

### 5.1 安装方式总览

|   方式    |     适合人群      |
| :-------: | :---------------: |
|  Desktop  |       新手        |
| Portable  | Windows 普通用户  |
| comfy-cli | 开发者 / 长期使用 |

### 5.2 几种方式

1、方式一：Desktop 版(推荐新手)

```
1.特点：
-内置 Python
-即下即用

2.流程：
-下载官方 Desktop
-解压
-启动
```

2、方式二：Windows Portable

```
适合 Windows + NVIDIA。
-解压
-运行 run_nvidia_gpu.bat
-自动启动浏览器
```

3、方式三：comfy-cli(官方推荐)

```
1、指令
pip install comfy-cli
comfy install
comfy launch

2、优点
节点 / 模型管理方便
干净
可脚本化
```

## 六 界面与基础操作

### 6.1 核心界面区域

```
画布（Node Graph）
左侧 Templates
右上角 Queue / Run
History（历史记录）
```

### 6.2 基础操作

```
右键空白处 → 添加节点
拖动端口 → 连接节点
Delete → 删除节点
Ctrl + S → 保存 Workflow
```

### 6.3 Queue 的概念

```
ComfyUI 是“任务队列”模型
每次 Run = 加入队列
支持多任务
适合批量生成
```
