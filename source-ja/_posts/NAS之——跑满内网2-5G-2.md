## 二 硬件分析

### 2.1 NAS

```
-4盘位及以上，如 DS920+、DS1522+ 等
-网口必须支持 2.5G 或可扩展 2.5G 网卡
```

### 2.2 硬盘

```
7200 RPM HDD 顺序读写约 150~200 MB/s/块
2.5G 内网理论带宽 ≈ 312 MB/s
单块 HDD 无法跑满 2.5G → 需要多盘并行/RAID或SSD Cache
```

### 2.3 PC

```
2.5G 网卡 + SSD 或 HDD
最新驱动，关闭省电模式
Jumbo Frame 9000(可选)
```

### 2.4 网络设备

```
-2.5G Switch 或 NAS ↔ PC 直连
-CAT6 或 CAT6A 网线
```

## 三 RAID 与存储优化

### 3.1 RAID类型选择

```
-RAID0 → 多盘顺序带宽叠加，性能最佳
-RAID10 → 性能 + 冗余，4盘总顺序读写 ≈ 300~400 MB/s → 可满跑 2.5G
-RAID5/6 → 安全性高，性能略低，但 4盘也能接近 2.5G 内网带宽
```

### 3.2 SSD Cache(可选)

```
-提升写入性能和随机 I/O
-对大文件传输有帮助，可充分利用 2.5G 内网
```

### 3.3 多线程 I/O

```
SMB3/NFS 多线程传输
避免单线程受单盘带宽限制
```

## 四 NAS 设置

### 4.1 网络

```
DSM → 控制面板 → 网络 → 2.5G 网口启用
链路聚合可选(提高多客户端总带宽)
```

### 4.2 Jumbo Frame

```
NAS + PC 都设置 MTU 9000
提升大文件传输效率
```

### 4.3 文件服务

```
SMB3 多线程或 NFS
可开启异步写入提高吞吐
```

### 4.4 节能关闭

```
DSM → 硬件与电源 → 网卡节能 → 关闭
避免传输中速度波动
```

## 五 PC 设置

```
-2.5G 网卡 + SSD/HDD
-最新驱动
-关闭省电模式 / Energy Efficient Ethernet
-Jumbo Frame MTU 9000
-多线程文件传输（≥4线程）
```

## 六 性能预期

### 6.1 列表

|      配置       |   顺序读写   |               内网利用率               |
| :-------------: | :----------: | :------------------------------------: |
|  4块 HDD RAID0  | 600~800 MB/s |       满速 2.5G 内网(≈312 MB/s)        |
| 4块 HDD RAID10  | 300~400 MB/s |           接近满速 2.5G 内网           |
| 4块 HDD RAID5/6 | 200~300 MB/s |       多线程拷贝可满跑 2.5G 内网       |
|    单块 HDD     | 150~200 MB/s | 单线程无法跑满 2.5G → 多线程可接近满速 |
| HDD + SSD Cache | 读写性能提升 |         单客户端接近 2.5G 满速         |

### 6.2 核心结论

```
单盘 HDD 无法跑满 2.5G，4盘 RAID10 或 RAID0 + 多线程拷贝即可稳定满速
```

## 七 总结

```
-RAID / 多盘 → 利用顺序带宽叠加，避免单盘限制
-多线程 SMB3 / NFS → 确保单客户端跑满 2.5G
-SSD Cache 可选 → 提升写入性能、减少瓶颈
-NAS + PC Jumbo Frame 9000 → 提升大文件传输效率
```
