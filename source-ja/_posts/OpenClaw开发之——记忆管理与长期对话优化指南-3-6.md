## 二 OpenClaw 的记忆

### 2.1 一句话理解

```
OpenClaw 的记忆 ≠ 永久存储，而是“总结 + 压缩 + 再利用”
```

### 2.2 核心目标

```
让 AI：
-记住关键内容
-忽略无关信息
-长期保持上下文一致性
```

## 四 记忆能力本质

### 4.1 Memory ≠ 存数据库

```
1.错误理解：
AI 会永久记住所有内容

2.正确理解：
AI 通过“总结”来记忆

3.本质流程：
对话 → 压缩 → 生成摘要 → 作为上下文继续使用
```

### 4.2 两种记忆方式

```
1.短期记忆（Session Memory）
-当前对话上下文
-自动保留最近内容
-开启：Hooks → 选择 session-memory

2.长期记忆（Memory Bank）
-定期总结对话
-保存关键知识
-下次继续使用
```

## 五 自动总结

### 5.1 手动总结

```
1.输入：总结我们刚才的对话重点

2.输出：
- 需求
- 关键决策
- 未完成事项
```

###  5.2 定期自动总结

```
实现方式：
-Prompt 触发
-定时任务（cron / Windows）
-Hooks 自动执行
```

### 5.3 示例(自动记忆)

```
1.输入：
每 10 轮对话总结一次，并保存为 memory.txt

2.输出逻辑：
-提取关键内容
-去除重复信息
-结构化输出
-写入文件
```

## 六 实现方式

### 6.1 文件存储(最简单)

```
1.存储结构

2.示例：

- 完成 OpenClaw 部署
- 配置 Ollama + Gemini
- 未完成：自动化脚本
```

```
1.Linux：
crontab -e
0 */2 * * * openclaw chat "总结最近对话并更新 memory.md"

任务计划程序 → 定时执行
```

```
-形成日报 / 周报
```

## 八 常见问题

### 8.1 AI“失忆”

```
原因：上下文太长被截断
解决：定期总结 + 压缩
```

### 8.2 记忆混乱

```
原因：没有结构化总结
建议：使用固定模板
```

### 8.3 无法长期保存

```
原因：只存在对话中
解决：写入文件 / 数据库
```

## 九 最佳实践

```
一、记忆系统 = 三层结构
1. Session Memory（短期）
2. Summary（压缩）
3. Storage（存储）

二、推荐组合：
OpenClaw + Summary + Shell + 文件存储
```
