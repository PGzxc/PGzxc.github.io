## 二 关于被封

### 2.1 为什么会被封？

1-本质原因

```
你的流量“太像代理”
```

2-常见触发点：

|    风险点    |     说明      |
| :----------: | :-----------: |
|   明文协议   |   容易识别    |
|   固定特征   | 流量特征明显  |
|  IP 被标记   | VPS IP 被识别 |
| 异常连接频率 | 请求过于频繁  |

3-简单理解：

```
正常用户：浏览网页
异常用户：大量加密 + 高频请求 + 特征明显
```

### 2.2 为什么 Reality 能抗封？

1-核心原理：

```
你的流量 → 伪装成 HTTPS → 混入正常流量
```

2-访问来看

```
表面看：访问 Cloudflare / Microsoft
实际：进入你的服务器
```

3-关键点

|  技术   |    作用    |
| :-----: | :--------: |
| TLS伪装 | 像正常网站 |
| SNI伪装 | 像访问大厂 |
| 无证书  |   更隐蔽   |

## 三 实施步骤

### 3.1 Reality 最优配置(实战优化)

1-推荐配置(优化版)

```
"realitySettings": {
  "show": false,
  "dest": "www.cloudflare.com:443",
  "serverNames": [
    "www.cloudflare.com",
    "www.microsoft.com"
  ],
  "privateKey": "你的key",
  "shortIds": [
    "6ba85179e30d4fc2"
  ]
}
```

2-优化点说明:

|    参数     |  优化建议  |
| :---------: | :--------: |
|    dest     | 用大厂域名 |
| serverNames |    多个    |
|  shortIds   |    随机    |
|    show     |   false    |

### 3.2 SNI 伪装策略(非常重要)

```
1.推荐伪装域名：

- www.cloudflare.com
- www.microsoft.com
- www.apple.com

2. 原则：

-大流量网站
-全球 CDN
-正常访问频率高

3.不建议：
- 小网站
- 冷门域名
```

### 3.3 端口选择策略

1-推荐端口：

| 端口 | 推荐  |
| :--: | :---: |
| 443  | 5颗星 |
| 8443 | 4颗星 |
| 2053 | 3颗星 |

2-原因：

```
443 = HTTPS标准端口（最隐蔽）
```

### 3.4 多节点容灾(关键)

```
1.说明：
不要只用一个 VPS！

2.推荐结构：
HK（主）
JP（备用）
US（备用）

3.在 Clash 中：

fallback:
  - HK
  - JP
  - US

4.效果：
-一个挂了自动切
-提高稳定性
```

### 3.5 IP 风险控制

```
一、原因：
VPS IP 会被标记

二、解决方案：

2.1 多 IP：
- 多服务器
- 定期切换

2.2 更换节点：
被封就换 IP

2.3 避免滥用：
不要：
- 大量爬虫
- 高频请求
```

### 3.6 连接行为优化(很多人忽略)

```
1.降低异常特征：

-不要频繁断开重连
-不要同时开太多连接
-避免短时间大量请求

2.类比:
-正常用户：浏览网页
-异常用户：疯狂刷新1000次
```
