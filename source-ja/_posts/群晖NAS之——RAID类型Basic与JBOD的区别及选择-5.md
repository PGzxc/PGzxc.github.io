## 一 概述

```
本文介绍：
 1.容易混肴的RAID模式：Basic与JBOD
 2.两者看起来都“没保护、容量大”，但其实区别挺大的，尤其是坏盘后的数据命运和使用场景
 3.一句话：Basic 像电脑里插多块独立硬盘，JBOD 像把多块硬盘硬绑成一个超大U盘
```

<!--more-->

## 二 核心区别

### 2.1 对比

|     项目      |                     Basic 模式                      |                          JBOD 模式                           |
| :-----------: | :-------------------------------------------------: | :----------------------------------------------------------: |
| 硬盘数量要求  |           每池1块硬盘(多硬盘=多个独立池)            |              至少1块，多块起(多硬盘合成1个大池)              |
|  容量怎么算   |        每个硬盘容量独立(比如2×10T=2个10T卷)         |           所有硬盘容量直接加总(2×10T=20T一个大卷)            |
|  坏一盘后果   |         只丢坏的那块硬盘数据，其他硬盘没事          | 几乎全丢！(坏盘数据丢光；如果坏的是第一盘，可能整个JBOD池全废) |
|   读写性能    |         每个卷独立，IO不干扰，适合随机读写          |            顺序写多时碎片化严重，频繁读写磨损不均            |
| 以后扩容/升级 | 超级方便：单盘Basic可升级到RAID 1/5/SHR(加盘直接转) |         基本不行：JBOD不能直接加盘或升级，得删池重建         |
| 硬盘大小混用  |           完美支持(不同大小硬盘各玩各的)            |        支持，但浪费小盘空间(系统从第一盘开始顺序填满)        |
|   管理方式    |        每个硬盘一个存储池/卷，共享文件夹分开        |            一个大存储池/卷，所有数据混在一个卷里             |
|   适用场景    |            电影/下载/备份多硬盘独立管理             |             想“假装一个超大硬盘”，但不怕丢全家桶             |

### 2.2 风险

```
JBOD 坏盘风险远高于Basic。

群晖官方和社区都说，JBOD的元数据(分区表)主要放第一块盘，坏第一盘整个阵列可能认不出；
Basic每块硬盘独立，坏一块只影响那一块。
很多人用JBOD后悔过：“以为容量大，结果一坏全没了”。
```

## 三 怎么选择

### 3.1 选 Basic 的场景(推荐大多数人)

```
1.有多块不同大小硬盘（比如8T+10T+18T）。
2.想硬盘坏了只丢一部分数据(电影丢了能重下，重要东西另备份)。
3.以后可能加盘升级到SHR/RAID(Basic单盘起步，超级灵活)。
4.管理上喜欢每个硬盘独立卷(比如一个放电影、一个放照片、一个放下载)。
5.日常使用：存大文件、看片、备份，性能够用。
```

### 3.2 选 JBOD 的场景(很少人用，不建议)

```
1.硬盘大小差不多，想“一个大卷”统一管理(电脑映射一个盘符就全看到)。
2.纯备份用，且有外部多份备份(比如增量备份到JBOD，不在意单盘坏)。
3.极致容量追求，不care任何保护(但RAID 0其实更香，有速度加成)。
4.频繁顺序大文件写入(JBOD碎片少点)，但实际测试也没大优势。
```
