## 一 概述

```
本文介绍：
 1.群晖 NAS 在 Windows 上映射磁盘问题
 2.网络改变变(WiFi 重连、路由器重启、IP 晃动)，映射的Z盘、Y盘就红叉消失了，还得手动重新映射
 3.每次都得输入密码、选盘符
 4.最后用群晖官方的 Synology Assistant 来映射
```

<!--more-->

## 二 问题原因

### 2.1 NAS IP 地址改变了(最常见于“网络变化”)

```
-路由器用 DHCP 自动分配 IP，如果没固定，NAS 偶尔就会换 IP（比如重启路由器后）。
-你映射时用了 \\192.168.1.100\共享文件夹，IP 一变路径就失效，映射直接“阵亡”。
```

### 2.2 Windows 映射驱动器不会“智能重连”

```
1.勾了“登录时重新连接”，它只在开机登录那一刻检查网络。
2.后面 WiFi 断一下重连、笔记本切换网络、或者闲置太久，Windows 就不管了，直接断开
（默认 15 分钟空闲就可能掉)。

3.快速启动功能、凭证缓存问题也会加剧。
```

### 2.3 其他原因

```
SMB 协议不匹配、凭证冲突、盘符重复
```

## 三 为什么 Synology Assistant 映射更稳？

### 3.1 原因

```
1.它用群晖主机名（或 MAC 自动发现）而不是裸 IP，IP 变了也自动找到 NAS。
2.内置“登录时重新连接”逻辑，比 Windows 原生映射更聪明。
3.自动处理凭证缓存，减少“红叉”概率。
4.一键选共享文件夹、选盘符，傻瓜式操作，新手 3 分钟搞定。
```

### 3.2 前提

```
1.要开启 SMB 服务（DSM 控制面板 → 文件服务 → SMB → 启用 SMB 服务），
2.防火墙别挡 445 端口，用户有对应共享文件夹读写权限。
```

## 四 解决流程

### 4.1 第1步：NAS 侧打好基础

```
1.登录DSM—>控制面板—>网络—>网络接口—>编辑 LAN—>选使用静态 IP，
填好 IP、子网掩码、网关、DNS。保存。
（或者路由器里给 NAS MAC 做 IP 保留，双保险。）

2.控制面板 → 文件服务 → SMB → 勾选启用 SMB 服务，
高级设置里开启用 WS-Discovery 和 Windows 网络发现。

3.控制面板—>共享文件夹—>选你要映射的文件夹—>编辑—>权限—>给映射专用账号(别用 admin)读写权限。
```

### 4.2 第2步：下载 & 安装 Synology Assistant(Windows 电脑上)

```
1.打开浏览器，搜“Synology 下载中心” 或直接去 https://www.synology.com/zh-cn/support/download
2.选你的机型 DS1821+（或通用版），下载 Synology Assistant（Windows 版）。
3.安装，桌面会出现小图标。
```

### 4.3 第3步：用 Synology Assistant 一键映射

1-通过`\\Synology1821`映射

```
1.Win资源管理器，输入\\Synology1821并回车
2.在弹窗列表，选择要映射的共享文件夹
3.在共享文件夹上右键—>映射网络驱动器，
4.输入你的群晖用户名 + 密码，确定无误后点击完成
5.网络位置，显示所有已映射的共享文件夹
6.以后开机、网络重连，共享文件夹不用出现红叉
```

2-通过Synology Assistant映射

```
1.双击打开 Synology Assistant，它会自动搜索局域网里的群晖设备。
2.看到你的 DS1821+(显示主机名或 IP)，点选它。
3.上方工具栏出现 映射驱动器（或“Map Drive”）按钮，点它！
4.弹出向导—>输入你的群晖用户名 + 密码(映射专用账号最好)—>下一步。
5.列表显示 NAS 上你有权限的共享文件夹 → 勾选你要映射的（比如 photo、video、home、data 等，可多选映射多个盘）—>下一步。
6.选盘符（推荐 Z:、Y: 这种靠后的）—> 重要：勾选“登录时重新连接”—>下一步。
7.预览总结，没问题点 完成（Finish）。
8.打开“此电脑”，Z: 盘就静静地躺在那儿了！

以后开机、网络重连，Assistant 会帮你自动重连映射，不用再管红叉。
```
