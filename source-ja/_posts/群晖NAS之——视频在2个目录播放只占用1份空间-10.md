## 一 概述

```
本文介绍：同一个视频文件在两个位置都能播放，但只占用一份硬盘空间。常见有3种方法
 1.硬链接(Hard Link) 最推荐
 2.符号链接(Symbolic Link/软链接)
 3.文件快捷方式(DSM File Station)
```

<!--more-->

## 二 场景需求

```
1.例如你的 NAS 有两个目录：

-/video/电影
-/video/4K电影

2.某个电影希望：
-在 电影目录可以看到
-在 4K电影目录也能看到
-但 只占用一份硬盘空间
```

## 三 实现方法对比

|         方法          | 是否占用额外空间 | 是否稳定 | 推荐程度 |
| :-------------------: | :--------------: | :------: | :------: |
|   硬链接 Hard Link    |       不占       |  4颗星   |   推荐   |
| 软链接 Symbolic Link  |       不占       |  3颗星   |   可用   |
| File Station 快捷方式 |       不占       |  2颗星   |  不推荐  |

最佳方案：硬链接

## 四 方法1：硬链接(最推荐)

### 4.1 原理

```
1.硬链接相当于 同一个文件的多个入口：

原文件
  ├─路径A
  └─路径B

但 实际硬盘只存一份数据。

2.特点：
-不占额外空间
-删除其中一个不影响另一个
-视频软件都能识别
```

### 4.2 操作步骤

1-开启 SSH

```
1.DSM 控制面板：

控制面板
→ 终端机和SNMP
→ 启用 SSH
```

2-连接 NAS(终端或三方如MobaXterm)

```
1.Windows / Mac：
ssh admin@NAS_IP

2.例如
ssh admin@192.168.1.10
```

3-创建硬链接(要求：同一个池且最好不要出现换行)

```
假设：

1.原视频：
/volume1/video/电影/流浪地球.mp4

2.想出现在：
/volume1/video/科幻电影/

3.执行：
ln /volume1/video/电影/流浪地球.mp4 \
   /volume1/video/科幻电影/流浪地球.mp4

4.完成后：
电影/
   流浪地球.mp4
科幻电影/
   流浪地球.mp4

两个目录都能播放。
```

4-验证是否占用空间

```
执行：ls -li
如果 inode 相同说明是同一个文件。
```

### 4.3 注意事项

```
1、硬链接 必须在同一存储卷，即不能跨卷
volume1 
volume2 

2、删除文件规则
删除一个硬链接 → 文件仍存在
删除所有链接 → 文件才会删除
```

## 五 方法2：软链接(Symbolic Link)

### 5.1 说明

```
软链接类似 Windows 的 快捷方式。
```

### 5.2 创建方法及示例

```
1.创建方法：
ln -s 原文件 目标路径

2.示例：
ln -s /volume1/video/电影/流浪地球.mp4 \
      /volume1/video/科幻电影/流浪地球.mp4
```

### 5.3 特点

```
1.优点
-不占空间
-创建简单

2.缺点
-删除原文件会失效
-某些播放器识别不好
```

## 六 方法3：File Station 快捷方式

### 6.1 创建

```
在 DSM 文件管理器中：

右键文件
→ 创建快捷方式
```

### 6.2 特点

```
但这种方式：
-只在 DSM 里有效
-很多播放器无法识别

因此 不适合视频库管理。
```
