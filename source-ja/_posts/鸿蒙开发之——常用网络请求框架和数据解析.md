## 一 概述

```
在鸿蒙应用开发中，
网络能力主要围绕 系统原生API展开，同时在 ArkTS/JS体系下，数据解析方式也更加偏向JSON+类型映射。
与Android/iOS 不同，鸿蒙目前第三方网络库生态较少，以官方能力为主。
```

<!--more-->

## 二 鸿蒙常用网络请求方式

### 2.1 HTTP / HTTPS(主流)

```
适用于：
-接口数据请求
-登录/列表/配置下发
-业务数据同步
```

### 2.2 WebSocket(实时通信)

```
适用于：
-即时消息
-实时推送
-设备状态同步
```

## 三 鸿蒙常用网络库/网络能力

### 3.1 @ohos.net.http(主流官方方案)

```
1-定位：鸿蒙官方 HTTP 网络模块
2-适用语言：ArkTS / JS

3-核心能力：
-HTTP/HTTPS 请求
-GET/POST/PUT/DELETE
-Header/参数配置
-Promise/Callback 风格
-支持超时、证书配置

4-特点：
-官方维护，稳定可靠
-与系统权限、生命周期结合紧密
-生态最成熟

5-说明：
鸿蒙应用网络请求首选方案
```

### 3.2 fetch API(轻量级)

```
1-来源：Web 标准 API
2-支持环境：ArkTS/JS

3-特点：
-语法简单
-Promise 风格
-上手成本低

4-不足：
-功能相对简单
-拦截、统一处理能力有限

5-说明：
适合简单请求/小型应用
```

### 3.3  WebSocket(@ohos.net.webSocket)

```
1-定位：官方 WebSocket 模块

2-使用场景：
-聊天
-实时推送
-IoT 状态通信
```

### 3.4 第三方网络库(较少)

```
目前鸿蒙生态中：
-类 Retrofit / OkHttp 的成熟库较少
-大多基于官方 HTTP 再封装
-企业项目通常自研网络封装层

鸿蒙网络层以“自封装”为主
```

## 四 鸿蒙常用数据解析方式

### 4.1 JSON(绝对主流)

```
鸿蒙前后端交互几乎全部使用 JSON：
{
  "code": 0,
  "data": {},
  "msg": "success"
}
```

### 3.2 JSON.parse/JSON.stringify(基础方式)

```
1-最常用、最基础：
const obj = JSON.parse(response);

2-特点：
-无依赖
-简单直接
-无类型约束
```

### 3.3 ArkTS 类型映射(推荐)

```
1-在 ArkTS 中，通常结合 interface / class 做数据约束：

interface BaseResponse<T> {
  code: number;
  msg: string;
  data: T;
}


2-优势：
-可读性好
-代码规范
-便于维护

3-说明：
工程中最常见做法
```

### 3.4 第三方 JSON 解析库(少量)

```
1-部分项目会使用：
-fast-json（JS 方案）
-手写映射工具

2-说明：
但总体不如 Android/iOS 成熟。
```

### 3.5 XML 解析(较少)

```
使用场景：
-老系统
-特殊设备协议
```

## 四 网络请求 + 解析的典型封装思路(鸿蒙)

### 4.1 分层结构(推荐)

```
UI (Page / Ability)
   ↓
ViewModel
   ↓
Repository
   ↓
HttpService（@ohos.net.http 封装）
```

### 4.2 统一返回结构

```
interface BaseResponse<T> {
  code: number;
  msg: string;
  data: T;
}
```

### 4.3 统一错误处理

```
-网络异常
-HTTP 状态码
-业务错误码
-登录失效
```

### 4.4 Header / Token 统一注入

```
-请求前统一注入 Token
-统一 User-Agent
-日志开关控制
```

## 五 鸿蒙与 Android/iOS 网络差异

|   对比项   |    鸿蒙    |   Android    |     iOS     |
| :--------: | :--------: | :----------: | :---------: |
|  底层网络  |  官方模块  |    OkHttp    | URLSession  |
|  主流封装  |   自封装   |   Retrofit   |  Alamofire  |
| JSON 解析  | JSON.parse | Gson / Moshi |   Codable   |
| 第三方生态 |    较少    |   非常成熟   |    成熟     |
| 网络层模式 |  官方优先  |   社区优先   | 官方 + 社区 |
