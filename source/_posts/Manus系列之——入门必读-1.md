## 二 Manus 是什么？

### 2.1 官方定义

```
Manus 是一个能够自主规划、执行并交付完整结果的通用 AI Agent
```

### 2.2 核心区别

|     类型     | 传统 AI（ChatGPT类） |           Manus           |
| :----------: | :------------------: | :-----------------------: |
|   工作方式   |     问一步答一步     |    一次任务 → 自动完成    |
| 是否执行任务 |      只输出文本      |         真正执行          |
|  是否有环境  |          无          |  虚拟电脑（可运行代码）   |
|   输出结果   |         文本         | 文件 / 网站 / 报告 / 应用 |

### 2.3  一句话总结

```
Manus = 能帮你“做完事情”的 AI，而不是只“回答问题”的 AI

Manus = AI Agent（不是聊天机器人）
```

## 三 核心能力(官方能力拆解)

### 3.1 自主执行(最核心)

```
1.Manus 会自动：
- 拆解任务
- 制定步骤
- 执行流程
- 输出最终成果

2. 例如：
你说：帮我分析50家公司并生成报告

Manus 会：
-自动拆任务
-搜索数据
-分析
-生成报告文件

无需你一步步提示
```

### 3.2 虚拟工作环境(关键能力)

```
1.Manus 运行在一个完整环境中：
- 有浏览器
- 有文件系统
- 可以运行代码
- 可以安装软件

2.官方描述：
Manus 在一个带互联网和持久文件系统的 sandbox 环境中运行 

3.这意味着：

- 能写代码并运行
- 能处理 Excel / PDF
- 能部署网站
```

### 3.3 多任务执行(端到端)

```
1.支持完整工作流：
- 数据分析
- 网站开发
- 文档生成
- 自动化流程
- API 调用

2.示例（官网）：
- 自动生成 PPT
- 创建网站
- 批量处理数据 
```

### 3.4 Wide Research(多 Agent 并行)

```
1.Manus 的核心黑科技：

使用多个 AI Agent 并行处理任务 

2.特点：
- 可同时分析几十/几百个对象
- 每个任务独立上下文
- 不会“越做越差”

3.对比传统 AI：
- ChatGPT：顺序处理（越多越乱）
- Manus：并行处理（稳定输出）
```

### 3.5 输出结果不是“文本”，而是“成果”

```
1.Manus可以输出：
- 报告（PDF/Word）
- 数据表
- 网站
- 分析图
- 代码项目

2.官方强调：

Manus delivers complete work products, not just responses 
```

## 四 典型使用场景(官方案例)

```
1.数据类
- 清洗数据
- 转换 Excel / CSV
- 生成结构化表格

2.开发类
- 构建网站
- 开发工具
- 自动生成 API

3.研究类
- 市场分析
- 公司调研
- 竞品分析

4.办公类
- 写简历
- 写邮件
- 做 PPT

5.官网案例：
- Build website
- Create slides
- Organize data 
```

## 五 计费方式(官方)

```
1.Manus 使用：
Credits(积分制)

2.特点：
- 按任务复杂度消耗
- 支持 Free / Pro / Team 

3.优化建议（官方）：
- 描述越清晰 → 越省钱
- 合并任务 → 减少调用
```

## 六 Manus vs ChatGPT(官方思路总结)

|    对比项    | ChatGPT  |  Manus   |
| :----------: | :------: | :------: |
|     类型     | 对话 AI  | AI Agent |
|   是否执行   |  不执行  |   执行   |
| 是否自动规划 |    否    |    是    |
|     输出     |   文本   |   成果   |
|   使用方式   | 多轮对话 | 一次任务 |

## 七 本篇总结

```
Manus 的本质是：
从“问答 AI” → “执行型 AI” 的进化
```
