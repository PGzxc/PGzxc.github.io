## 二 开始使用 Manus

```
1.官网入口：
https://manus.im/

2.注册 / 登录
目前 Manus 采用：
- 邀请制（Waitlist）
- 或通过已有账号登录

3.官方入口路径：
- 点击 Get Started
- 填写申请或直接登录

4.注意（官方说明）：
- Manus 仍处于逐步开放阶段
- 不同用户权限可能不同
```

## 三 界面结构(官方 UI 拆解)—主要区域

进入 Manus 后，核心界面非常简单：

### 3.1 输入区(最重要)

```
你只需要输入一个任务：
 -帮我分析AI市场并生成报告
 -Manus 会自动完成全部流程
```

### 3.2 任务执行区(Agent 工作区)

```
1.这里会显示：
- 任务拆解过程
- 执行步骤
- 工具调用
- 运行日志

2.类似：
Step1: 搜索数据
Step2: 整理信息
Step3: 生成报告

3.官方强调：
用户可以实时查看 Agent 的执行过程
```

### 3.3 输出结果区

```
1.最终结果可能是：
- 文件（PDF / Excel）
- 网页
- 数据表
- 代码项目

2.可直接下载或查看
```

## 五 提示词(Prompt)最佳实践

Manus 不需要复杂 Prompt，但需要：清晰 + 具体

### 5.1 错误示例

```
1.示例：
帮我分析一下

2.问题：
- 不知道分析什么
- 输出不可控
```

### 5.2 正确示例

```
1.示例：
分析2025年AI行业发展趋势，重点关注大模型和Agent方向，并生成报告

2.优点：
- 有范围
- 有重点
- 有输出形式
```

### 5.3 官方建议总结

```
- 明确目标（做什么）
- 指定范围（数据/领域）
- 指定输出（报告/网站/表格）
```

## 六 任务执行机制—Manus 背后流程

```
1.Task Planning(任务规划)
自动拆解任务

2.Tool Usage(工具调用)

可能使用：
- 浏览器
- 代码执行
- 文件系统

3.Execution(执行)
逐步完成任务

4.Delivery(交付)
输出最终成果

官方强调：用户不需要手动控制流程
```

## 七 如何提高成功率(实战经验 + 官方建议融合)

### 7.1  1次只做一类任务

```
1.好：生成AI报告
2.不好：写报告 + 做网站 + 分析数据
```

### 7.2 明确输出格式

```
生成PDF报告 或：输出Excel表格
```

### 7.3 给上下文(非常关键)

```
目标用户是开发者

会显著提升质量
```

### 7.4 文件任务要说明用途

```
用于商业分析
```
