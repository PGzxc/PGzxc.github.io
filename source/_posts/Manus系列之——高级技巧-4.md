## 二 Prompt 设计(核心中的核心)

### 2.1  官方原则总结

```
Manus 不需要复杂 Prompt，但需要“结构化任务描述”
```

### 2.2 最优 Prompt 结构(通用模板)

```
1.通用模板
目标（做什么）
范围（处理什么数据/内容）
步骤（需要完成哪些事情）
输出（最终结果形式）

2.标准示例：
帮我分析AI编程工具市场，
范围：Cursor、Copilot、Codeium
步骤：
-功能对比
-优缺点分析
-使用场景
输出：生成一份结构化报告（Markdown格式）

3.优点：
- Agent 更容易理解任务
- 执行更稳定
- 结果更可控

4.低质量 Prompt：
分析一下AI工具

问题：
- 无范围
- 无输出
- 无结构
```

### 2.3 强制控制输出(非常关键)

```
1.Manus 默认“自由发挥”，所以你要约束它

2.示例:
输出要求：
- 使用表格
- 包含结论
- 控制在1000字以内

3.常用输出控制：
- 格式：Markdown / PDF / Excel
- 风格：简洁 / 专业 / 技术向
- 长度：字数限制
- 结构：标题 / 表格 / 列表
```

### 2.4 上下文增强(提高质量)

```
目标用户：开发者
用途：技术选型

结果会明显更专业
```

## 三 成本优化(Credits 使用技巧)

Manus 使用 Credits 计费(官方机制)

### 3.1 成本消耗来源

```
主要消耗在：

- 任务复杂度
- 执行步骤数量
- 外部数据调用
- 多 Agent 并行
```

### 3.2 降低成本的 5 个方法(实测有效)

```
方法1：减少任务范围
-错误：调研所有AI产品
-正确：调研3个主流AI编程工具

方法2：合并相关步骤
-错误：分3次执行任务
-正确：一次性完成分析 + 输出

方法3：避免重复执行
不要反复让 Manus 做同一件事

方法4：限制输出规模
控制在500字以内

方法5：优先结构化输出

表格 > 长文本（更省资源）
```

## 四 提速技巧(让任务更快完成)

### 4.1 控制任务复杂度

```
Manus 是“执行型 AI”，不是秒回

慢任务：
分析100家公司 + 生成报告

优化：
先分析10家公司
分批执行更快
```

### 4.2 避免无关步骤

```
错误：
分析 + 设计UI + 写代码 + 部署

原因：太重
```

### 4.3 明确执行路径

```
只需要数据分析，不需要网页

可减少执行步骤
```

### 4.4  使用 Wide Research(合理使用)

```
官方能力：并行 Agent

适合：
- 批量调研
- 数据收集

不适合：
- 小任务（浪费资源）
```

## 五 稳定性优化(避免翻车)

### 5.1 拆任务(最重要)

```
大任务拆小：

错误：
做一个完整电商网站 + 后端 + 数据分析

正确：
-生成网站结构
-再生成功能
-再优化 UI
```

### 5.2 增加约束条件

```
不要生成多余内容
```

### 5.3 明确边界

```
只分析提供的数据，不要联网搜索
```

### 5.4 遇到失败怎么办？

```
官方建议：
- 重新描述任务
- 减少复杂度
- 增加约束
```

## 六 高手技巧(进阶玩法)

### 6.1 技巧1：让 Manus 先“思考再执行”

```
先给出执行计划，再开始任务

可以提升成功率
```

### 6.2  技巧2：生成可复用模板

```
生成一个数据分析模板，下次可复用
```

### 6.3 技巧3：结合本地数据

```
 上传文件 + AI分析 = 最强组合
```

### 6.4 技巧4：结果再加工

```
Manus 输出后：再优化这份报告，让它更专业
```
