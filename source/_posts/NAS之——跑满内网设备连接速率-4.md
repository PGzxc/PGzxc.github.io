## 一 概述

```
本文介绍：NAS跑满情况下，其他设备(比如电脑/pad/手机/电视)呢，通过网线或WiFi连接
```

<!--more-->

## 二 NAS 网络速率

```
NAS 可以跑满 2.5 Gbps最大速率，也就是大约 312.5 MB/s(注意：1 B = 8 bit)
实际速率会略低，通常在 280–300 MB/s 左右，受硬盘、RAID、协议开销影响。
```

## 三 其他设备连接情况

### 3.1 有线连接(网线)

```
1、千兆网线(1 GbE)
-最大带宽 1 Gbps ≈ 125 MB/s。
-如果电脑只有千兆网口，NAS 2.5 Gbps 也跑不满，实际吞吐就是 约 120 MB/s。

2、2.5 GbE/5 GbE/10 GbE 网卡
-需要设备和交换机同时支持。
-例如：电脑和 NAS 都是 2.5 GbE，交换机也是 2.5 GbE，就能接近 300 MB/s。
-如果设备是 10 GbE，则理论可跑满 NAS，前提硬盘能支撑。
```

### 3.2 Wi-Fi 连接

Wi-Fi 最大速率取决于标准

|  Wi-Fi 标准  | 理论速率  |   实际吞吐    |
| :----------: | :-------: | :-----------: |
| Wi-Fi 5 (ac) | 1.3 Gbps  | 400–600 Mbps  |
| Wi-Fi 6 (ax) | 2.4 Gbps  | 800–1200 Mbps |
|   Wi-Fi 6E   | 3.5 Gbps+ |   1–2 Gbps    |

### 3.3 结论

```
-电脑/手机/Pad通过 Wi-Fi，很难跑满2.5Gbps，通常低于1 Gbps，除非使用 Wi-Fi 6/6E 并靠近路由器。
-Wi-Fi 距离远、障碍多时速率会下降。
```

## 四 网络拓扑对速度的影响

```
-如果NAS 直连电脑(2.5 GbE)，可接近满速。
-如果 NAS → 交换机 → 设备：交换机每端口速率需匹配，不能低于 NAS 或设备的速率，否则成为瓶颈。
-多设备同时访问：2.5 Gbps 会被分摊，单台设备速度可能不达满速。
```

## 五  总结

```
1、NAS 2.5 Gbps 是上限，能跑满前提：
-设备网口也支持 2.5 GbE。
-网线、交换机都支持 2.5 GbE。

2、千兆设备(电脑/电视/Pad)只能跑到 约 1 Gbps。

3、Wi-Fi 设备：
-受限于 Wi-Fi 标准和信号强度，一般跑不到 2.5 Gbps。
-Wi-Fi 6 或 6E 设备在理想条件下可接近 1–1.2 Gbps。
```
