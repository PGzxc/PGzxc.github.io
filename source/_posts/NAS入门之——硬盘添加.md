## 一 概述

```
本文介绍：
 - NAS添加硬盘如何操作？
 - 依次添加和一起添加？
```

<!--more-->

## 二 明确前提条件

```
在添加硬盘前，请先确认：
-NAS 型号（例如群晖、威联通、西部数据、绿联等）
-当前磁盘状态（是否已有阵列 / 是否已有数据）
-新硬盘是否为全新（未格式化 / 无分区）
-当前使用的 RAID 模式（RAID 1、5、6、SHR、JBOD 等）

不同系统操作界面不同，但逻辑类似，以下为通用流程。
```

## 三 添加方式对比

|   方式   |                        说明                         |               适用场景               |                 注意事项                  |
| :------: | :-------------------------------------------------: | :----------------------------------: | :---------------------------------------: |
| 依次添加 | 先使用 1 块或几块硬盘建立 NAS，后续再插入新硬盘扩容 | 先用现有盘建阵列，后期预算增加时扩展 | 要求支持“在线扩容”（如群晖 SHR、RAID5/6） |
| 一起添加 |             一次性插入所有硬盘再初始化              |         新建 NAS 或重建阵列          | 容量最大化、性能最好，但所有盘会被格式化  |

## 四 操作流程（以群晖/威联通为例）

### 4.1 情况 1：全新 NAS，一起添加硬盘

```
1、适合：新机器首次安装

2、步骤
2-1、插入所有硬盘

-插入所有计划使用的硬盘；
-开机后进入 NAS 管理界面（DSM、QTS、My Cloud OS）。

2-2、初始化系统

-系统会提示创建存储空间；
-选择所有硬盘；
-设置 RAID 类型（如 SHR、RAID5）；
-选择文件系统（Btrfs 或 EXT4）；
-格式化 → 初始化 → 创建存储池。

2-3、创建卷和共享文件夹

-存储池创建后，建立卷（Volume）；
-在卷中创建共享文件夹；
-NAS 可立即使用。

3、优缺点
优点：阵列最优、容量最大、结构清晰。
缺点：所有硬盘数据会被清空。
```

### 4.2 情况 2：已有阵列，依次添加新硬盘

```
1、适合：NAS 已在用，想扩容。

2、步骤
2-1、插入新硬盘

-NAS 通常自动检测；
-“存储管理器” → “存储池” → “管理”。

2-2、扩充存储池

-选择“添加硬盘”或“扩展存储池”；
-选择要加入的空盘；
-系统会进行 RAID 重新计算（Resync）。

2-3、等待重建

-重建过程较慢（数小时至数十小时）；
-NAS 仍可使用，但性能会下降。

2-4、扩展卷

-重建完成后，在“卷管理”中选择“扩展容量”。

3、优缺点
优点：无需重装系统，数据不丢失。
缺点：扩容时间长；RAID 结构固定，无法随意更改 RAID 类型（如从 RAID1 → RAID5 需重建）。
```

### 4.3 情况 3：额外插盘但不加入现有阵列

```
1、说明：想把新硬盘作为独立磁盘使用。

2、步骤
-插入新硬盘；
-在“存储管理器”中选择“新建存储池”；
-单独设置该硬盘（Basic 或 JBOD 模式）；
-创建新卷，用作独立存储。

3、适合：想把某块盘专门用于备份、监控录像、下载目录等。
4、注意：这类盘不享受 RAID 容错。
```

## 五 添加后检查步骤

|    检查项目    |            操作路径             |
| :------------: | :-----------------------------: |
| SMART 健康状态 | 存储管理器 → HDD/SSD → 健康信息 |
|   RAID 状态    | 存储池 → 状态（正常 / 重建中）  |
|     卷容量     |      存储卷 → 容量及使用率      |
|   共享文件夹   |      控制面板 → 共享文件夹      |

## 六 建议总结

|            需求            |                     推荐方式                     |
| :------------------------: | :----------------------------------------------: |
|    新建 NAS（首次安装）    |                 一起添加全部硬盘                 |
|      已有 NAS，想扩容      |                依次添加并在线扩容                |
| 不同用途的硬盘（如备份盘） |                  单独创建存储池                  |
|    想灵活调整 RAID 结构    | SHR 模式（群晖）或 RAID Migration 功能（威联通） |

## 七 额外建议

```
-扩容前务必备份数据（可临时备份到外置盘或云端）；
-扩容期间不要断电；
-若要更换 RAID 类型（如 RAID1 → RAID5），请备份后重建阵列；
-WD NAS / 绿联 NAS 通常采用简化界面，可在“存储管理 → 创建卷”中完成全部操作。
```
