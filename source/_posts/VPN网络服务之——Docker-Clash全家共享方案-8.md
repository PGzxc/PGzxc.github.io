## 一 概述

```
完成后你可以：

-NAS 上运行代理（24小时在线）
-手机 / 电视 / 平板自动走代理
-不用每台设备单独配置
-Emby / Jellyfin 解锁资源
```

## 二 全局共享方案

### 2.1 为什么要做“全局代理”？

```
1.现在的痛点：

当前状态：
-电脑 有代理
-手机 没代理
-电视 没代理
-NAS 没代理

结果：
- 设备要一个个配置
- 很麻烦
- 有些设备根本不支持代理

2. 解决方案：
把代理放在“网络中心”（NAS / 路由器）
```

### 2.2 整体架构(核心理解)

```
1.整体架构：

所有设备（手机/电视/电脑）
          ↓
        路由器 / NAS
          ↓
      Clash（Docker）
          ↓
      Xray（VPS）
          ↓
         互联网
         
2.核心思路：
让 NAS 变成“代理网关”     
```

### 2.3 方案选择(很重要)

1-方案对比

|    方案    | 难度 |  推荐  |
| :--------: | :--: | :----: |
| 路由器刷机 |  高  | 不推荐 |
| NAS Docker |  中  |  推荐  |
| 单设备代理 |  低  | 不推荐 |

2-推荐

```
NAS + Docker + Clash
```

## 三 部署实战

### 3.1 在 NAS 上部署 Clash

1-使用镜像(推荐)—Clash Meta Docker

```
docker run -d \
  --name clash \
  -p 7890:7890 \
  -p 7891:7891 \
  -p 9090:9090 \
  -v /volume1/docker/clash:/root/.config/clash \
  --restart always \
  dreamacro/clash
```

2-端口说明

| 端口 |   作用    |
| :--: | :-------: |
| 7890 | HTTP代理  |
| 7891 | SOCKS代理 |
| 9090 | 控制面板  |

### 3.2 导入配置

```
把之前的 Clash 配置文件：config.yaml
放到：/volume1/docker/clash/
然后重启容器：docker restart clash
```

### 3.3 开启局域网访问(关键)

```
1.修改配置：
allow-lan: true
bind-address: 0.0.0.0

2.作用：
局域网设备可以访问 NAS 代理
```

## 四 应用

### 4.1 让手机 / 电视使用代理

```
方法一：手动设置（简单）

1.手机 WiFi 设置：
代理：手动
IP：NAS地址（如 192.168.1.100）
端口：7890

2.效果：
手机直接走 NAS 代理

方法二：透明代理(进阶)

1.目标：无需设置 → 自动走代理
2.实现方式：
- 路由器转发
- iptables
- TProxy（高级）
```

### 4.2 NAS 应用走代理—Docker 容器走代理

```
1.添加环境变量：
-e http_proxy=http://192.168.1.100:7890
-e https_proxy=http://192.168.1.100:7890

2.应用场景：
- Docker 拉镜像
- Emby 插件
- 自动下载工具
```

### 4.3 Emby / Jellyfin 解锁

```
1. 原理：
媒体服务 → NAS → Clash → VPS → 外网

2. 效果：
-海外资源可访问
-元数据正常加载
-插件可用
```

### 4.4 进阶：Clash Web 面板

```
1.访问：
http://NAS-IP:9090

2.可以：
- 切换节点
- 查看日志
- 测速
```

## 五 常见问题

### 5.1 手机连不上

```
原因：

- 没开 allow-lan
- 防火墙拦截
```

### 5.2 Docker 不走代理

```
原因：

- 没设置环境变量
```

### 5.3 速度慢

```
原因：

- NAS 性能不足
- 节点慢
```
