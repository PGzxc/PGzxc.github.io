## 二 网络访问

### 2.1 网络访问烦恼

```
1. 很多人第一次接触“代理”或“VPN”，都是因为这些日常小烦恼：

-GitHub 克隆超级慢，甚至失败
-Docker 拉镜像卡住半天不动
-某些 API 突然访问不了
-同一个网站，不同地区看到的内容完全不一样

2.本质就一个问题：

-你的网络路径不稳定，或者被限制了。
-流量从你电脑出去，要经过运营商、国际出口，最后才能到目标服务器。
-这中间任何一环出问题，都会卡顿或直接失败。
```

### 2.2 互联网访问的真实路径

```
1. 正常直连时，流量是这样走的

你（本地设备）
   ↓
运营商（电信/移动/联通）
   ↓
国际出口
   ↓
目标服务器（Google、GitHub、Docker Hub 等）

2. 常见问题出在：
-国际出口拥堵（晚高峰特别明显）
-路由绕远路（延迟高）
-网络策略限制（直接被挡住）
```

### 2.3 代理的本质

```
1.代理本质
代理 = 找一台“中转服务器”帮你访问目标网站
简单流程：text你 → 代理服务器 → 目标网站

2.便于理解
-直连就像自己跑去国外超市买东西，又远又累还可能被拦。
-代理就像找个“海外代购”，他帮你买好再寄回来，省心多了。
```

## 三 Proxy 与 VPN

### 3.1 Proxy(代理)是什么？

```
1.定义
Proxy 是“应用层转发”，只处理你指定的部分流量，不是全部。

2.工作方式
 -浏览器（或某个软件） → Proxy → 目标网站
 -举例：浏览器走代理，但游戏、系统更新、微信可能还是直连。

3. 常见协议
 -HTTP Proxy：只支持网页，比较老。
 -SOCKS5：通用性强，推荐新手用（支持更多软件）。
 -VLESS/VMess/Trojan：新一代协议，更安全、隐蔽性好，常用于Xray/V2Ray。

4. 优点
 -灵活：想让哪个软件走代理就配哪个。
 -性能好：只绕需要绕的流量。
 -支持精细分流（国内直连、国外走代理）。

5. 缺点
 -需要自己配规则，新手容易搞混。
 -不是所有软件都支持。

小Tips：日常开发时，用Clash或v2rayN这类客户端，就能轻松管理规则。
```

### 3.2 VPN 是什么？

```
1.定义:
VPN（虚拟专用网络）就是建立一条加密隧道，把你的流量安全包裹起来。

2.工作方式
你 → 加密隧道 → VPN服务器 → 互联网所有流量（包括浏览器、App、系统更新）都会走这条隧道。

3.常见协议（2026年推荐顺序）
 -WireGuard：速度快、配置简单，现代首选。
 -OpenVPN：成熟稳定，但稍重一点。
 -IPsec 等企业级协议（个人用得少）。
 
4.优点
-全局生效，一键开启，所有软件自动走。
-配置简单，新手友好。
-加密更彻底，隐私保护更好。

5.缺点
-不够灵活（所有流量都绕，可能国内网站也变慢）。
-速度有时会打点折扣。
```

### 3.3 VPN vs Proxy

1-表格

|   项目   |  VPN（全局隧道）   |   Proxy（局部代理）   |
| :------: | :----------------: | :-------------------: |
| 作用范围 |  全局（所有流量）  | 局部（指定软件/规则） |
|  灵活性  |        较低        |   很高（精细分流）    |
| 配置难度 |   低（一键为主）   |   中等（需配规则）    |
|   性能   |   一般（全绕路）   |    更好（按需绕）     |
| 适合场景 | 日常上网、企业安全 |  开发测试、精细操作   |

2-核心结论

```
现在很少人纯用一种，而是VPN负责传输 + Proxy负责分流，两者结合最香。
```

## 四 现代主流玩法

### 4.1 实际架构

```
你的电脑
   ↓
本地客户端（Clash / v2rayN / Nekobox 等）
   ↓
协议处理（WireGuard / Xray / VLESS 等）
   ↓
远程服务器（VPS，如DMIT上的节点）
   ↓
互联网（国内直连 + 国外走代理）
```

### 4.2 分流

```
1.概念
分流 = 让不同流量走不同路径，避免所有流量都绕远路。

2.简单例子：
 -国内网站（baidu.com、taobao.com）→ 直连（快！）
 -国外网站（google.com、github.com）→ 走代理

3.规则写法类似这样（Clash风格）：
YAMLrules:
  - DOMAIN-SUFFIX,baidu.com,DIRECT
  - DOMAIN-SUFFIX,google.com,Proxy

4.类比：
国内流量走自家高速，国外流量走专用通道，既快又稳。
```

### 4.3 开发者视角

```
1.最简单解决方案：
用本地代理（比如结合Cliproxy或自建节点）。

2.终端里敲一行命令就行：
Bashexport http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890
或者用 proxychains、git config 等工具。

3.本质就是：
CLI工具 → 本地代理端口 → 远程服务器 → 互联网。
这样GitHub、Docker、npm等开发工具就能顺畅跑起来，日常工作效率直接起飞。
```
