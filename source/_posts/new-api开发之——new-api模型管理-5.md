## 二 new-api模型

### 2.1 什么是“模型管理”？

1-在 New API 中：

```
模型 = 用户看到的名字（抽象层）
渠道 = 实际调用的上游 API
```

2-举个例子：

```
1.用户调用：
"model": "gpt-4o"

2.但实际：
 -可能走 OpenAI Key1
 -可能走 OpenAI Key2
 -甚至走代理 / 镜像

3.核心思想：
模型是统一入口，渠道是实际执行
```

### 2.2 模型的本质

1-模型 ≠ 真实模型：

```
New API 中的模型其实是：一个“逻辑名称(别名)”
```

2-举例：

| 模型名（对用户） |   实际调用    |
| :--------------: | :-----------: |
|      gpt-4o      | OpenAI GPT-4o |
|     fast-gpt     |    GPT-3.5    |
|     cheap-ai     |   本地模型    |

3-这意味着：

```
你可以“自定义模型品牌”！
```

## 三 模型管理

### 3.1 模型添加方式

```
1.模型不是单独添加的，而是：在渠道中定义

2.操作路径：
渠道管理 → 编辑渠道 → 模型列表

3.示例配置

gpt-4o
gpt-3.5-turbo
claude-3-opus

这些模型：
- 会显示给用户
- 可被 API 调用
```

### 3.2 模型映射(高级玩法)

```
1.什么是映射？
把一个模型名映射到另一个模型

2. 示例
用户调用：gpt-4
实际执行：gpt-4o

作用：
- 兼容旧接口
- 控制成本
- 动态切换模型

3.实战案例:
gpt-4 → gpt-4o（降成本）
gpt-3.5 → 本地模型（0成本）
```

### 3.3 模型倍率(赚钱核心)

1-什么是倍率？

```
售价 = 成本 × 倍率
```

2-示例

|  模型  | 成本 | 倍率 | 用户价格 |
| :----: | :--: | :--: | :------: |
| GPT-4o |  1   |  2   |    2     |
| Claude |  1   | 1.5  |   1.5    |

3-说明：

```
这就是你的利润来源
```

### 3.4 模型定价策略(实战)

1-方案1：基础盈利

|  模型   | 倍率 |
| :-----: | :--: |
| GPT-3.5 |  2   |
|  GPT-4  | 2~3  |

2-方案2：分用户等级

| 用户 | 倍率 |
| :--: | :--: |
| 免费 |  3   |
| 付费 | 1.5  |
| VIP  | 1.2  |

3-方案三：差异化模型

```
cheap-ai → 低价（吸引用户）
pro-ai → 高价（赚钱）
```

### 3.5 模型分层策略(非常重要)

1-为什么要分层？

```
不同请求成本不同！
```

2-推荐分层:

| 层级 |        模型        |   用途   |
| :--: | :----------------: | :------: |
|  低  | GPT-3.5 / 本地模型 | 简单问答 |
|  中  |       GPT-4o       | 普通用户 |
|  高  |   Claude / GPT-4   | 高端用户 |

3-实战策略

```
1.实战策略
简单问题 → 低模型
复杂问题 → 高模型

2.可结合：
- 用户等级
- Prompt 长度
- 使用场景
```

## 四 进阶

### 4.1 多模型调度(进阶)

```
1.什么是调度？
根据规则选择模型

2.示例

if 用户是 free → GPT-3.5
if 用户是 pro → GPT-4o
if 用户是 vip → Claude

3.高级玩法

- 自动判断问题复杂度
- 自动切换模型
- 成本优化
```

### 4.2 自定义模型(品牌化)

```
1.你可以这样做：
super-ai
fast-ai
cheap-ai
pro-assistant

2.实际：
super-ai → GPT-4
cheap-ai → 本地模型

3.意义:
用户只认“你的品牌”，不关心底层模型
```
