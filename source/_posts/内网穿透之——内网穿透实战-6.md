## 二 相关概念

### 2.1 解决什么问题？

```
1.很多人搭了 NAS 或家庭服务器后都会遇到同一个问题：
 -在家可以访问（局域网 OK）
 -出门就访问不了
 -DDNS 配好了但不稳定
 -运营商没有公网 IP（最常见）

2.本质问题只有一个：

你的设备在内网，外网无法直接访问
```

### 2.2 典型家庭网络结构

```
手机（外网）
   ↓
公网IP / CGNAT
   ↓
路由器（NAT）
   ↓
NAS / 服务器（192.168.x.x）

NAS 默认是“隐藏在内网”的设备
```

## 三 远程访问的 4 种主流方案

### 3.1 方案1：NAS官方DDNS(最简单)

```
1、适合：
新手 / 临时使用

2、常见设备：
Synology DiskStation
QNAP TS Series

3、思路：
NAS → 自动绑定域名 → 外网访问

4、示例：
https://yourname.synology.me

5、优点：
-不需要额外服务器
-一键配置

6、缺点：
-依赖公网IP（很多家庭没有）
-安全性一般
-不适合复杂服务（Docker / API）
```

### 3.2 方案2：路由器端口映射(Port Forwarding)

```
1.适合：有公网IP用户

2.示例：
ASUS RT-AC66U B1

3.原理：
公网IP:5000 → NAS:5000

4.操作步骤：
-登录路由器后台
-设置端口转发
-指向 NAS 内网 IP

5.优点：
-速度快（直连）
-延迟低

6.缺点：
-极度不安全（暴露端口）
-需要公网IP
-容易被扫描攻击
```

### 3.3 方案3：内网穿透(推荐方案)

```
1.适合：
没有公网IP / 运营商 CGNAT

2.推荐工具：frp

3.架构：
NAS（内网） → frpc → VPS（公网服务器） → 手机访问

4.实战架构（推荐）
       公网VPS（服务器）
              ↓
        frps（服务端）
              ↓
        加密隧道
              ↓
      NAS / 家庭服务器（frpc）

5.适用场景
你可以远程访问：
-NAS 文件管理
-Docker 服务
-Home Assistant
-私人云盘
-Jellyfin / Emby 影音库
-Git / CI 服务

6.优点
-无需公网IP
-稳定
-可控性高
-支持多服务映射

7.缺点
需要一台 VPS
初始配置稍复杂
```

### 3.4 方案4：VPN(WireGuard / Tailscale)

```
1.适合：
长期使用 / 安全优先

2.WireGuard 方式
-类似“虚拟局域网”
-手机 = 直接进入家里内网

3.结合设备：
-WireGuard
-Tailscale

4.优点：
-极高安全性
-像在同一局域网
-访问 NAS 就像在家一样

5.缺点：
-需要设备支持
-某些网络环境速度受限
```

## 四 推荐组合架构(最实用)

### 4.1 最佳实践(个人/家庭)

```
1.组合方案：

Tailscale（VPN打底）
        +
frp（服务暴露）

2.组合优势：
-VPN：安全访问 NAS
-frp：对外提供服务（网站/API）
```

### 4.2 企业级升级版

```
Zero Trust（零信任）
   +
VPN Mesh
   +
反向代理（Nginx）
   +
内网穿透
```

## 六 安全建议

### 6.1 不要做

```
直接暴露 NAS 管理端口
用弱密码
开 SSH 默认端口
```

### 6.2 推荐做

```
VPN优先访问
frp + token认证
HTTPS加密
限制IP访问
```
