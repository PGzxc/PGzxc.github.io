## 一 概述

```
本文介绍：
 1.不同品牌NAS迁移到群晖
 2.硬盘迁移到群晖(带数据和无数据)
 3.借助移动硬盘
```

## 二 迁移前整理

### 2.1 设备清单

```
1.nas: 群晖1821+(8盘位) 、mycloud ex2(2盘位)
2.nas硬盘：
 -18T(在1821+第1个盘符，shr模式，容量将用完，大部分电影+小部分重要数据)
 -2x10T(空，单独硬盘未放入nas)
 -1个8T在mycloud ex2中(3.4T左右电影数据) basic模式
 -1个8T(在1821+第2个盘符，basic模式容量快用完，大部分电影)
 
3.移动硬盘：1个4T(未使用)+1个4T(快用完，电影数据)+1个5T(快用完，开发资源)+1个4T(快用完，电影数据)
4.电脑：win台式机
5.路由器：asus ac66ub1
6.网线：千兆
```

### 2.2 要求

```
 1-硬盘空间利用率高
 2-兼顾安全与数据备份
 3-操作时间短
 4-方便下次硬盘更换(比如8T换成18T以及更大)
 5-数据不丢失
 6-nas设备能否更换(8盘位能否换成4盘位(偏向)或5盘位、8盘位)
 
 将现在的nas整理一下，将nas硬盘同一放到1821+设备下管理(不再使用mycloud ex2)，如何操作以及耗时多少
```

## 三 方案对比(从简单到复杂)

### 3.1 硬盘直迁(最快，但限同品牌群晖)

```
关1.机拔硬盘，按顺序插新NAS，开机DSM认出旧存储池(SHR/Basic都支持)。
2.优点：零复制，数据秒到。
3.缺点：
-WD My Cloud EX2硬盘格式不同(ext4或WD专有)，1821+认不了，得先拷数据。
-群晖间迁才完美。
-降盘位（8→4）也行，但bay少就迁部分硬盘。
```

### 3.2 网络复制(最通用，最常用)-目前没通过

```
1.File Station挂载老NAS共享(CIFS/SMB)，右键复制到新NAS。
2.优点：不拔盘，边用边拷，校验方便。
3.缺点：千兆网限速，29TB数据可能1-2天。
4.适合WD EX2：挂载Public共享，拷temp或其他文件夹。
```

### 3.3 Shared Folder Sync 或 Hyper Backup(定时/增量)-目前没通过

```
1.装套件，设任务自动拉数据，支持版本历史。
2.优点：增量快，以后变化自动同步。
3.缺点：设置稍多。
```

### 3.4 Migration Assistant(群晖→群晖专用)

```
系统设置+套件全迁，适合同品牌升级。
WD不支持。
```

## 四 实际选择(从WD EX2迁1821+)

```
1.先备份重要数据到移动硬盘。
2.EX2的Public等文件夹，用File Station挂载SMB，内部复制到1821+新Basic卷。
3.拷完拔EX2硬盘，插1821+格式化加池。
4.移动硬盘USB直连1821+，用USB Copy包一键导入，速度飞起。
```

## 五 总结

```
1.千兆网下，内部复制最快；电脑中转最慢。
2.先小文件夹测试速度和权限。
3.迁完跑scrub检查硬盘健康。
4.以后换NAS(4/5/8)，硬盘直迁或网络迁都行，群晖间最丝滑。
```
