## 二 什么是 RAID

```
1. 概念
RAID(Redundant Array of Independent Disks)
即 独立磁盘冗余阵列，是通过多块硬盘组合，

2.作用：提高：存储容量/数据安全/读写性能

3.说明：NAS 中 RAID 是 存储池的核心结构。
```

## 三 传统RAID类型

### 3.1 传统RAID

```
Basic：单盘独立用，没任何保护。硬盘坏了，那盘数据全丢。适合测试、临时存垃圾数据，或者你压根不怕丢。
JBOD：几块盘简单串起来，总容量加起来最大，但也没保护，一坏丢一块。
RAID 0：速度最快（条带），容量全用，但零保护，一坏全池完蛋。极少人用，除非纯追求速度+有备份。
RAID 1：镜像，两盘互拷，一坏还能用，但容量只剩一块盘大小。
RAID 5：3盘起，能坏1盘，容量减1块。
RAID 6：4盘起，能坏2盘，容量减2块。
RAID 10：速度+保护结合，但至少4盘，容量减半。
```

### 3.2 说明

```
传统RAID要求硬盘大小尽量一样，不然小盘拖后腿，浪费空间；
扩容也麻烦，得全换大盘。
```

## 四 群晖常见RAID类型(SHR)

```
1.SHR(或叫SHR-1)：
-至少2盘，能坏1盘，相当于RAID 5的保护，但硬盘大小混着用也没问题！
-它把每块盘切成小块，自动优化，浪费空间最小。

2.SHR-2：至少4盘，能坏2盘，像RAID 6，但一样支持混盘。
3.单盘SHR：没保护，跟Basic差不多，但以后加盘就能变SHR-1。
```

## 六 RAID 类型选择建议

```
1.电影/照片多，数据能重下 → Basic 或单盘SHR(简单，容量最大)
2.重要数据，想防1盘坏 → 
 -SHR(或SHR-1)，混盘友好，
 -以后换18T→更大盘超方便(Storage Manager里替换硬盘，系统自动重建，边用边修)
3.超级怕丢(双备份不够) → SHR-2
4.追求极致速度、无备份需求 → RAID 0(不推荐)
```
