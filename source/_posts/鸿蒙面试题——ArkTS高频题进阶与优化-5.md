## 二 进阶与优化(仅供参考)

### 2.1 异步编程模型？

```
-Promise：链式调用
-async/await：同步风格
-Worker：多线程
```

### 2.2 NAPI 的作用？

```
-提供 ArkTS 与 C/C++、Rust 的桥梁
-场景：性能关键模块、多媒体解码、算法库
```

### 2.3 ArkTS 性能优化方法？

```
-避免频繁 setState
-合理使用 ForEach 渲染大列表（分页/懒加载）
-Worker 分担耗时任务
```

### 2.4 内存泄漏场景？

```
-未释放的全局引用
-循环引用导致 GC 无法回收
-事件监听器未解绑
```

### 2.5 启动优化方法？

```
-减少首页初始化逻辑
-异步加载非关键资源
-预加载必要数据
```

### 2.6 测试方式？

```
-单元测试：@ohos.unitTest
-UI 测试：模拟器 + 自动化测试框架
```

### 2.7 打包与签名流程？

```
HAP 构建 → 签名（使用证书） → 打包为 AppGallery 发布包
```
