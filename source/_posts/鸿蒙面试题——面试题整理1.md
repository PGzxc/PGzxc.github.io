#### 2.2.3 [页面的生命周期][02]

```
页面生命周期，即被@Entry装饰的组件生命周期，提供以下生命周期接口

* onPageShow：页面每次显示时触发一次，包括路由过程、应用进入前台等场景
* onPageHide：页面每次隐藏时触发一次，包括路由过程、应用进入后台等场景
* onBackPress：当用户点击返回按钮时触发
```

### 2.3 [鸿蒙APP和Android的区别][03]

```
1.设计不一样
鸿蒙：为多设备互联设计（手机、平板、车机、家电通用）
安卓：只为手机为主的单设备设计

2.架构不一样
鸿蒙：Stage 模型，核心是 UIAbility
安卓：传统组件模型，核心是 Activity、Service

3.开发语言 & UI
鸿蒙：ArkTS + ArkUI（声明式）
安卓：Kotlin/Java + XML/Compose

4.跨设备能力
鸿蒙：系统自带分布式软总线，应用可直接跨设备流转、共享硬件
安卓：靠蓝牙 / WiFi，应用自己实现，无系统级协同

5.应用形态
鸿蒙：可安装 APP，也支持原子化服务（免安装即用）
安卓：只能安装 APK

6.底层与运行
鸿蒙：微内核，方舟编译，更流畅安全
安卓：Linux 内核，ART 虚拟机，长期易卡顿
```

### 2.4  [父子间传递数据方式有哪些][04]

```
说明：此问题涉及状态的同步(装饰器)及状态的管理(存储)

1.状态的同步
1. @State装饰器：组件内状态
2. @Prop装饰器：父子单向同步
3. @Link装饰器：父子双向同步
4. @Provide装饰器和@Consume装饰器：与后代组件双向同步
5. @Observed装饰器和ObjectLink装饰器：嵌套类对象属性变化

2.状态的管理(存储)
1. LocalStorage：页面级UI状态存储
2. AppStorage：应用全局的UI状态存储
3. PersistentStorage：持久化存储UI状态
4. Environment：设备环境查询

3.其他状态管理
1. @Watch装饰器：状态变量更新通知
```

### 2.5  [跨进程通信方式有哪些][05]

```
鸿蒙提供 6 种主流跨进程通信(IPC)方式，覆盖简单通知、RPC 调用、大数据传输、跨设备协同等全场景，
底层基于 Binder 驱动（单设备）+ 分布式软总线(跨设备)实现。

一 概念介绍
1.1、公共事件 CommonEvent（发布 - 订阅）
适用：跨应用 / 进程的异步通知、状态广播（简单数据）
特点：一对多、解耦、无需知道对方进程
场景：系统事件（安装 / 关机）、应用内全局通知
示例：
// 发布进程
commonEventManager.publish("EVENT_USER_LOGIN", { data: "uid_10086" });

// 订阅进程
const info = { events: ["EVENT_USER_LOGIN"] };
commonEventManager.createSubscriber(info, (err, subscriber) => {
  subscriber.on("EVENT_USER_LOGIN", (event) => {
    console.log("登录：", event.data);
  });
});

1.2、IPC/RPC(IDL 接口调用)
适用：跨进程方法调用、强交互、返回值（最常用）
原理：鸿蒙 IDL（类似 AIDL）定义接口 → 生成代理 → Binder 通信
特点：像本地调用一样跨进程、同步 / 异步、支持复杂类型
场景：UI ↔ 服务（Service/Extension）、系统能力调用
示例：
// 服务端（提供接口）
class IMyAbilityStub extends IMyAbility {
  add(a: number, b: number) { return a + b; }
}

// 客户端（代理调用）
let proxy: IMyAbility = ...; // 从SAMgr获取
let sum = proxy.add(2, 3); // 跨进程调用

1.3、MessageChannel（消息通道）
适用：同一设备内、频繁双向消息（点对点）
特点：全双工、异步、低延迟、自定义通道
场景：进程间实时消息、长连接交互
示例：
// 创建通道
const channel = new MessageChannel();
// 发送
channel.postMessage({ cmd: "play", url: "xxx.mp3" });
// 接收
channel.onMessage = (msg) => { console.log(msg); };

1.4、Want 启动传参（Intent 类似）
适用：启动 Ability 并传递简单数据
特点：随组件启动传递、一次性、无回调
场景：打开页面 / 服务并传参
示例：
const want = {
  deviceId: "",
  bundleName: "com.demo",
  abilityName: "PlayAbility",
  parameters: { songId: "1001" }
};
context.startAbility(want);

1.5、共享内存 SharedMemory
适用：超大数据（图片 / 视频 / 文件）、高性能、零拷贝
特点：多进程映射同块物理内存、无拷贝、极高吞吐
场景：相机预览、视频解码、大图传输
示例：
// 创建共享内存
const shm = sharedMemory.create("data_shm", 1024 * 1024);
// 写入
const buf = shm.getByteBuffer();
buf.write("大数据");
// 传递句柄给另一进程

1.6、分布式数据对象 DistributedDataObject
适用：跨设备进程数据同步（超级终端）
特点：自动跨设备同步、面向对象、冲突处理
场景：多端协同（手机→平板续播、智能家居）

二、 底层核心

- 单设备 IPC：Binder 驱动（一次拷贝、高性能）
- 跨设备 RPC：分布式软总线（统一通信、自动组网）
- 安全：全链路权限校验、能力访问控制

三、选型建议

- 简单通知 → CommonEvent
- 方法调用 → IPC/RPC（IDL）
- 实时消息 → MessageChannel
- 超大数据 → SharedMemory
- 跨设备协同 → DistributedDataObject
- 启动传参 → Want
```

### 方式对比（选型指南）

|        方式         |   性能   |  数据量  |   模型   | 跨设备 |      典型场景      |
| :-----------------: | :------: | :------: | :------: | :----: | :----------------: |
|   **CommonEvent**   |    中    |    小    |  一对多  | 不支持 |   通知、状态广播   |
|     **IPC/RPC**     |    高    |    中    |  点对点  |  支持  | 服务调用、方法交互 |
| **MessageChannel**  |    高    |    中    |  点对点  | 不支持 |  实时消息、长连接  |
|      **Want**       |    低    |    小    |  一次性  |  支持  |      启动传参      |
|  **SharedMemory**   | **极高** | **超大** |   共享   | 不支持 |    音视频、大图    |
| **DistributedData** |    中    |    中    | 多端同步 |  支持  |     跨设备协同     |

### 2.6 @BuilderParam与@Builder有什么不同

```
* @Builder装饰器：自定义构建函数
* @BuilderParam装饰器：该装饰器用于声明任意UI描述的一个元素，类似slot占位符
```

### 2.7 [UIAblity的启动流程][07]

```
UIAbility(鸿蒙Stage模型)启动流程，
分冷启动（首次）、热启动（已存在）、后台启动三类，核心是生命周期回调的有序触发。

一、标准前台冷启动（首次启动）-完整序列：

1.系统进程创建：
AMS 创建应用进程、初始化 ArkUI 运行时、加载 AbilityStageHuawei Developer
2.onCreate(want, launchParam)
实例创建、全局初始化（数据库、服务、参数解析）
只执行一次
onCreate(want: Want, launchParam: AbilityConstant.LaunchParam) {
  // 初始化、存want
}
3.onWindowStageCreate(windowStage)
创建窗口、加载主页面
onWindowStageCreate(windowStage: window.WindowStage) {
  windowStage.loadContent('pages/Index', (err, data) => {});
}
4.onForeground():进入前台、可见可交互
5.应用运行

二、热启动（实例已存在，再次拉起）
序列：
2.1 onNewWant(want):接收新参数、跳转指定页面
2.2 onForeground():回到前台

三、后台启动（startAbilityByCall）——序列：

3.1 onCreate
3.2 onBackground():直接后台、不创建窗口
3.3 后续切前台：onNewWant → onWindowStageCreate → onForeground

四、完整生命周期(含退出)——标准流程：

启动：onCreate → onWindowStageCreate → onForeground
前台→后台：onBackground
后台→前台：onNewWant → onForeground
退出销毁：onBackground → onWindowStageDestroy → onDestroy


五、关键回调说明
onCreate：实例创建、初始化、只一次
onWindowStageCreate：窗口创建、加载页面、只一次
onForeground：前台可见、恢复资源
onBackground：后台、释放资源
onNewWant：实例已存在、新意图、处理跳转
onWindowStageDestroy：窗口销毁
onDestroy：实例销毁、释放所有资源

六、启动模式影响
standard（默认）：每次新建实例 → 全流程
singleton：单实例 → 已存在则走 onNewWant → onForeground
specified：按 key 匹配 → 存在则复用、不存在新建
```
